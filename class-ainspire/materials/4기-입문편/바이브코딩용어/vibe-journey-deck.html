STEP 1 / 아이디어를 AI에게 맡기기
바이브코딩으로 시작합니다
나
사용자가 프롬프트 입력하면 영상 생성 API 호출해서 결과를 갤러리에 보여주는 웹앱 만들어줘.
AI
네, 시작하겠습니다. 먼저 프로젝트 구조부터 잡을게요. 입력 폼이 있는 메인 페이지, 영상 갤러리 페이지, 영상 생성 API를 호출하는 백엔드 라우트로 나누겠습니다. 이대로 파일을 생성하고 진행하겠습니다.
나
좋아, 그렇게 해줘.
AI
프로젝트 구조를 만들었습니다. app/, components/, lib/ 폴더와 기본 페이지를 생성했어요. 현재 컨텍스트에 파일 구조가 모두 올라와 있으니, 이어서 화면을 채우겠습니다.
바이브코딩
만들고 싶은 결과를 자연어로 설명하고 AI와 대화하며 코드를 완성해 나가는 개발 방식입니다.
에이전트
요청을 이해하고 파일 읽기, 코드 수정, 테스트 실행 같은 실제 개발 작업을 수행하는 AI입니다.
프롬프트
AI에게 전달하는 작업 지시문입니다. 무엇을 만들지, 어떤 조건인지 구체적으로 작성합니다.
컨텍스트
AI가 현재 참고할 수 있는 정보 전체입니다. 대화 내용, 코드 파일, 프로젝트 구조가 포함됩니다.
함께 알아둘 용어
프론트엔드사용자가 보는 화면 쪽입니다. 버튼·입력창·갤러리 등 눈에 보이는 부분입니다.
백엔드화면 뒤 서버 쪽입니다. 데이터 저장, 외부 서비스 호출을 처리합니다.
API프론트엔드·백엔드·외부 서비스를 잇는 연결 통로입니다.
서브에이전트큰 작업을 나눠 특정 역할을 맡아 처리하는 보조 에이전트입니다.
전역특정 파일이나 함수가 아닌 전체 프로젝트 범위에 적용되는 설정이나 값입니다.
STEP 2 / 개발 환경 켜기
IDE를 열고 서버를 실행합니다
CASE 1 — Claude / Antigravity / Codex (AI가 직접 실행)
AI
파일을 생성했습니다. 바로 npm install과 npm run dev를 실행하겠습니다. 완료되면 localhost:3000에서 확인하세요.
CASE 2 — VS Code / Cursor (직접 터미널 입력)
AI
IDE 하단 Terminal을 열고 아래 두 줄을 순서대로 입력하세요.

npm install
npm run dev

NPM이 패키지를 설치하고 개발 서버를 시작합니다. localhost:3000으로 확인하세요.
IDE
코드를 작성하고 수정하는 통합 개발 환경입니다. VS Code, Cursor가 대표적입니다. 이 프로젝트는 JavaScript(및 TypeScript) 기반으로, IDE 안에서 직접 실행하고 확인합니다.
Terminal
명령어를 입력해 컴퓨터와 개발 도구를 조작하는 창입니다.
CLI
Command Line Interface. 터미널에서 명령어를 입력해 프로그램을 사용하는 방식입니다.
NPM
JavaScript 패키지를 설치하고 관리하는 도구입니다. npm install, npm run dev 등을 실행합니다.
함께 알아둘 용어
JavaScript웹페이지에 동작을 더하는 프로그래밍 언어입니다. 이 프로젝트의 코드가 대부분 JavaScript(및 TypeScript)로 작성됩니다.
Localhost내 컴퓨터에서 실행 중인 개발 서버 주소입니다. 배포 전 브라우저에서 미리 확인합니다.
Port컴퓨터 안에서 실행 중인 프로그램을 구분하는 번호입니다. localhost:3000에서 3000이 포트입니다.
Bash / PowerShell터미널 안에서 실행되는 명령어 해석기입니다. Bash는 macOS/Linux, PowerShell은 Windows입니다.
STEP 2 / 환경 세팅 — 추가 용어 (참고)
프로젝트 폴더 안을 살펴봅니다
package.json
이 프로젝트가 어떤 패키지를 사용하는지, 어떤 명령어로 실행하는지 기록해둔 설정 파일입니다. npm run dev / build / start 같은 명령어와 설치된 패키지 목록이 여기에 있습니다.
node_modules
npm install을 실행하면 생기는 폴더입니다. package.json에 적힌 패키지들이 실제로 이 폴더에 저장됩니다. 직접 열거나 수정할 필요 없습니다.
Read
파일이나 데이터를 읽는 동작입니다. AI 에이전트가 기존 코드를 파악할 때 먼저 Read합니다. "현재 파일 구조를 읽고 설명해줘"처럼 요청할 수 있습니다.
Write
파일에 새 내용을 쓰는 동작입니다. 기능을 요청하면 에이전트가 파일을 새로 만들거나 내용을 Write합니다. Read와 짝을 이룹니다.
환경 관련 오류가 생겼을 때: 오류 메시지를 그대로 AI에게 보여주세요. "이 오류가 떴는데 어떻게 해야 해?"라고만 해도 됩니다. 오류 내용이 컨텍스트에 들어가면 AI가 원인을 찾고 수정까지 해줍니다.
STEP 3 / 화면 뼈대 만들기
HTML · CSS · JavaScript로 화면을 구성합니다
나
메인 페이지에 텍스트 입력창, 영상 생성 버튼, 갤러리 영역 만들어줘. React로 짜줘.
AI
app/page.tsx 파일을 작성하겠습니다. HTML로 구조를 잡고, CSS 모듈로 스타일을 입히고, 버튼 클릭 동작은 JavaScript로 처리합니다. 전부 하나의 React 컴포넌트 파일 안에 완성합니다.
나
좋아, 잘 됐어. Next.js 파일 구조는 그대로 유지하는 거지?
AI
네, app/ 라우터 구조 그대로입니다. Next.js 위에서 React가 돌아가는 형태라 기존 파일 배치를 바꾸지 않았습니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
HTML
웹페이지의 구조를 만드는 언어입니다. 입력창, 버튼, 갤러리 영역의 뼈대를 구성합니다.
CSS
웹페이지의 디자인을 담당하는 언어입니다. 색상, 여백, 배치, 반응형 화면을 설정합니다.
JavaScript
웹페이지에 동작을 추가하는 언어입니다. 버튼 클릭, API 요청, 화면 변경을 구현합니다.
React
화면을 컴포넌트 단위로 만들 수 있는 JavaScript 라이브러리입니다. 라이브러리란 자주 쓰는 기능을 미리 만들어 둔 코드 모음으로, 직접 만들지 않고 가져다 씁니다 — STEP 4에서 더 자세히 다룹니다.
함께 알아둘 용어
TypeScriptJavaScript에 타입 개념을 추가한 언어입니다. 데이터 형태를 미리 지정해 오류를 줄입니다.
Next.jsReact 기반 웹앱 프레임워크입니다. 페이지 구성, 라우팅, 서버 기능, 배포 최적화를 제공합니다.
Framework앱을 만들기 위한 기본 구조와 규칙을 제공하는 개발 도구입니다. Next.js가 대표적입니다.
STEP 3 계속 / 화면 구조 잡기
컴포넌트로 나누고 상태를 관리합니다
나
VideoForm이랑 VideoGallery를 별도 파일로 분리해줘. 입력 텍스트는 State로 관리하고, /gallery 경로도 따로 만들어줘.
AI
components/VideoForm.tsx, components/VideoGallery.tsx 두 Component로 분리했습니다. 입력 텍스트는 useState로 State 관리하고, app/gallery/page.tsx를 추가해 Routing을 설정했습니다.
나
헤더는 모든 페이지에 다 붙여야 해?
AI
app/layout.tsx에 한 번만 넣으면 됩니다. Layout은 모든 Page에 자동 적용되니 각 페이지에 헤더를 반복할 필요 없습니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
Component
화면을 구성하는 재사용 가능한 UI 조각입니다. VideoForm, VideoGallery 같은 단위로 관리합니다.
State
화면 안에서 변할 수 있는 상태값입니다. 사용자 입력, 로딩 여부처럼 변하는 값을 관리합니다.
Routing
사용자가 접속한 주소에 따라 어떤 페이지를 보여줄지 정하는 방식입니다.
함께 알아둘 용어
Props부모 컴포넌트가 자식 컴포넌트에 전달하는 데이터입니다. 같은 컴포넌트를 다르게 쓸 때 사용합니다.
Page사용자가 특정 주소로 접속했을 때 보게 되는 화면입니다. Next.js는 파일 구조로 경로를 구성합니다.
Layout여러 페이지에서 공통으로 사용하는 화면 구조입니다. app/layout.tsx에 헤더·푸터를 한 번 정의해 모든 페이지에 공유합니다.
헤더(Header)모든 페이지 위쪽에 공통으로 들어가는 영역입니다. 로고·메뉴 등을 포함하며 Layout에 한 번만 작성합니다.
STEP 4 / 기능 재료 설치
필요한 패키지와 라이브러리를 가져옵니다
나
supabase-js랑 lucide-react 설치해줘. package.json에 어디 들어가는 거야?
AI
터미널에서 실행하겠습니다.

npm install @supabase/supabase-js lucide-react

두 패키지 모두 배포 환경에서도 필요한 Dependency라 package.json의 "dependencies"에 기록됩니다. 이런 외부 코드 묶음을 Library라고 합니다.
나
typescript는 뭐야? 다른 칸에 들어가던데.
AI
TypeScript는 개발 중에만 필요한 DevDependency입니다. 코드를 작성할 때 오류를 잡아주는 도구라 실제 배포 환경에는 설치되지 않아 "devDependencies"로 분리됩니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
JavaScript 패키지
다른 개발자가 만들어 공개한 기능 묶음입니다. 직접 만들지 않고 NPM으로 설치해 가져다 씁니다. STEP 5의 Replicate SDK 설치, STEP 6의 supabase-js 설치가 모두 이 행위입니다.
Library
특정 기능을 쉽게 구현하도록 미리 만들어진 코드 모음입니다. 직접 구현하지 않고 가져다 씁니다. Framework과 달리 사용 방식을 강제하지 않습니다.
Dependency
앱이 실제로 동작할 때도 필요한 외부 패키지입니다. 배포 환경에도 설치됩니다. supabase-js, lucide-react 같은 것이 여기에 해당합니다.
DevDependency
개발 중에만 필요한 패키지입니다. 실제 배포 환경에는 설치되지 않습니다. typescript(타입 확인), eslint(코드 검사) 등이 여기에 해당합니다.
STEP 5 / 영상 생성 API · MCP 연결 (1/2)
API란 무엇이고, 어떻게 연결하나요
나
프론트에서 프롬프트 보내면 Replicate API로 영상 생성 요청하는 백엔드 라우트 만들어줘.
AI
app/api/generate/route.ts를 만들겠습니다. 프론트에서 HTTP Request를 이 Endpoint로 보내면, 서버가 Replicate API를 호출하고 영상 URL을 Response로 돌려주는 구조입니다.
나
프론트에서 직접 Replicate 호출하면 안 되는 이유가 있어?
AI
API Key가 브라우저에 노출됩니다. 백엔드에서 중계해야 키를 서버에서만 읽게 할 수 있습니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
API
프로그램끼리 데이터를 주고받기 위한 연결 방식입니다. 영상 생성 서비스를 코드에서 호출할 때 사용합니다.
HTTP
웹에서 데이터를 주고받기 위한 기본 통신 규칙입니다. 브라우저와 서버 사이의 언어입니다.
Request
클라이언트가 서버나 외부 서비스에 보내는 요청입니다. 프롬프트를 담아 API에 보냅니다.
Endpoint
API 요청을 보내는 구체적인 주소입니다. /api/generate처럼 기능별로 주소가 나뉩니다.
Response
Request에 대해 서버가 돌려주는 응답입니다. 생성된 영상 URL이나 오류 메시지를 담습니다.
STEP 5 계속 / 영상 생성 API · MCP 연결 (2/2)
요청 방식과 API Key를 알아야 합니다
나
Replicate SDK 써서 영상 생성 요청 코드 짜줘. 프롬프트는 JSON으로 담고 API Key는 환경변수에서 읽어오게.
AI
Replicate SDK로 작성하겠습니다. 영상 생성은 새 작업이라 POST 방식으로 보내고, 요청 body는 JSON 형식으로 prompt를 담습니다. API Key는 process.env에서 읽어옵니다.
나
갤러리 목록 조회할 때도 POST야?
AI
아니요, 조회는 GET입니다. POST는 새 데이터를 만들 때, GET은 기존 데이터를 읽을 때 씁니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
SDK
특정 서비스를 더 쉽게 사용하도록 제공되는 개발 도구 모음입니다. API를 직접 HTTP로 호출하는 대신, 서비스가 제공한 함수 한 줄로 간단히 호출할 수 있어 편합니다. 이 강의에서 Replicate SDK를 실제 사용합니다.
POST
새로운 데이터를 생성하거나 제출할 때 사용하는 HTTP 방식입니다. 영상 생성 요청에 사용합니다.
JSON
데이터를 주고받을 때 자주 사용하는 형식입니다. 키-값 쌍으로 구성된 텍스트 구조입니다.
API Key
외부 서비스를 사용할 때 발급받는 인증용 키입니다. 환경변수에 저장해 관리합니다.
GET
데이터를 조회할 때 사용하는 HTTP 방식입니다. 생성된 영상 목록을 불러올 때 씁니다.
STEP 5 / API 통신 흐름
Request → Response 한눈에 보기
사용자 (브라우저)
Request 발생

→
우리 백엔드 (Next.js)
/api/generate

→
외부 서비스 (Replicate API)
영상 생성 처리
프롬프트를 받아 AI 모델로 영상을 생성합니다. 완료되면 영상 URL을 JSON 형태의 Response로 돌려줍니다.
→
화면에 반영

백엔드가 받은 영상 URL을 브라우저로 전달합니다. 화면(갤러리)에 표시하고 Supabase DB에도 저장합니다.
GET
데이터를 가져올 때 사용합니다. URL에 정보가 포함됩니다.
GET /api/videos → 저장된 영상 목록 반환
POST
데이터를 생성하거나 제출할 때 사용합니다. 본문(body)에 JSON 데이터를 담습니다.
POST /api/generate + {"prompt":"..."} → 영상 생성 시작
JSON
요청과 응답 모두 JSON 형식으로 주고받습니다.
{"videoUrl":"https://...", "status":"done"}

Database에 영상 결과를 저장합니다
나
Supabase에 videos 테이블 만들어줘. video_url, prompt, user_id, created_at 컬럼 필요해. 영상 생성 후 row 저장하는 코드도.
AI
Supabase 대시보드에서 실행할 SQL과 클라이언트 코드를 함께 작성하겠습니다. videos Table에 영상 한 건이 Row 하나로 들어가고, video_url · prompt · user_id · created_at이 각각 Column이 됩니다. 생성 완료 후 supabase.from('videos').insert(...)로 저장합니다.
나
테이블 구조 바꾸고 싶으면 어떻게 해?
AI
Schema를 수정하면 됩니다. 컬럼 추가·삭제 SQL을 작성해드릴게요. 기존 Row 데이터는 그대로 유지됩니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
Database
앱에서 사용하는 데이터를 저장하는 공간입니다. 영상 URL, 사용자 정보 등을 보관합니다.
Supabase
데이터베이스, 로그인, 파일 저장 기능을 제공하는 백엔드 플랫폼입니다. 서버 없이 백엔드를 구현합니다.
Table
같은 종류의 데이터를 행과 열 구조로 저장하는 기본 단위입니다.
Row
테이블에 저장된 실제 데이터 한 줄입니다. 영상 하나, 사용자 한 명이 각각 하나의 Row입니다.
함께 알아둘 용어
Column테이블에서 세로 방향의 항목입니다. video_url, prompt 같은 데이터 속성을 표현합니다.
Schema데이터가 어떤 구조로 저장되어야 하는지 정의한 설계도입니다.
Relation테이블 사이의 관계입니다. 사용자(User)와 영상(Video)처럼 서로 연결된 데이터를 표현합니다.
STEP 6 보조 / 데이터베이스 심화 용어
DB 구조를 코드로 관리합니다
Prisma
코드에서 데이터베이스를 쉽게 다룰 수 있게 해주는 ORM 도구입니다. schema.prisma 파일로 구조를 정의합니다.
Migration
데이터베이스 구조 변경 이력을 관리하는 과정입니다. 테이블 추가, 컬럼 변경을 기록하고 적용합니다.
PostgreSQL
널리 사용되는 관계형 데이터베이스입니다. Supabase와 Neon이 PostgreSQL 기반 서비스를 제공합니다.
Neon: 클라우드 기반 PostgreSQL 서비스입니다. Prisma와 함께 사용하면 연결 문자열만으로 데이터베이스에 접근할 수 있습니다. Supabase가 통합 플랫폼이라면, Neon은 PostgreSQL DB에 집중한 서비스입니다.
STEP 7 / API Key 숨기고 로그인 붙이기
.env로 키를 숨기고, Auth로 로그인합니다
나
API 키들 .env 파일로 분리해줘. Supabase 이메일 로그인도 붙이고, 로그인한 사람만 영상 생성 가능하게.
AI
.env.local을 만들어 Secret Key들을 코드에서 분리하겠습니다. 이 파일은 .gitignore에 추가해 GitHub에 올라가지 않도록 처리합니다.

로그인은 Supabase Auth로 구현하고, 로그인 후 유지되는 Session을 미들웨어에서 확인해 비로그인 요청을 차단하겠습니다.
나
API 호출할 때 로그인 여부를 어떻게 확인해?
AI
Supabase Auth가 로그인 시 Token을 발급합니다. API 요청 헤더에 이 토큰을 담아 보내면 서버에서 권한을 검증합니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
왜 필요?
API Key를 코드에 직접 쓰거나 GitHub에 올리면 누구나 훔쳐 쓸 수 있습니다 — 과금·데이터 유출로 이어집니다. .env로 분리하고 .gitignore에 추가해 코드 밖에서 관리합니다. 로그인(Auth)이 없으면 누구나 API를 호출하거나 데이터에 접근할 수 있어 Auth + Token으로 접근을 제한합니다.
.env
환경변수를 저장하는 파일입니다. API Key, DB URL처럼 코드에 직접 쓰면 안 되는 값을 관리합니다.
Secret Key
외부에 공개되면 안 되는 중요한 인증 키입니다. 서버 환경변수로만 관리해야 합니다.
Auth
사용자가 누구인지 확인하는 인증 기능입니다. 로그인, 로그아웃, 회원가입에 사용합니다.
Session
사용자가 로그인한 상태를 유지하기 위한 정보입니다. 페이지를 이동해도 로그인 상태가 유지됩니다.
함께 알아둘 용어
Token사용자의 인증 상태나 권한을 확인하는 문자열입니다. API 요청에 함께 보내 권한을 증명합니다.
RLSRow Level Security. Supabase DB에서 사용자가 자기 데이터만 볼 수 있도록 행 단위 접근 권한을 설정합니다.
Environment VariablesAPI 키, DB 주소 같은 민감한 값을 코드와 분리해 관리하는 방식. .env 파일이 그 수단입니다.
STEP 8 / GitHub에 올리기
Git으로 저장하고 GitHub에 올립니다
나
GitHub에 올리려고. .gitignore 설정하고 첫 커밋 명령어 알려줘.
AI
.env.local과 node_modules를 .gitignore에 추가하겠습니다. 이후 순서대로 실행하세요.

git add . → git commit -m "init: project setup" → git push

Commit으로 변경 내용을 저장하고, Push로 GitHub에 올립니다.
나
기능 추가할 때마다 main에 직접 푸시해도 돼?
AI
Branch를 따로 파서 작업하는 걸 권장합니다. main 코드에 영향 없이 개발하고, 완성되면 합치는 방식입니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
Git
코드 변경 이력을 관리하는 버전 관리 도구입니다. 변경 내용을 저장하고 이전 상태로 되돌릴 수 있습니다.
GitHub
Git으로 관리하는 코드를 온라인에 저장하고 공유하는 플랫폼입니다. 배포 서비스와 연결해 사용합니다.
Commit
코드 변경 내용을 저장하는 단위입니다. 작업이 완료될 때마다 메시지와 함께 기록합니다.
Push
로컬 컴퓨터의 Commit을 GitHub 원격 저장소에 올리는 동작입니다.
함께 알아둘 용어
Repository프로젝트 코드와 변경 이력을 저장하는 공간입니다. GitHub에서 프로젝트별로 만들어 관리합니다.
Branch기존 코드에서 갈라져 나온 별도 작업 공간입니다. 메인 코드에 영향 없이 새 기능을 개발합니다.
Pull / Pull RequestPull은 원격 코드를 가져오는 동작, Pull Request는 변경 코드를 메인에 합치기 전 검토 요청입니다.
STEP 8 계속 / Vercel로 배포
GitHub과 연결하면 자동 배포가 됩니다
나
Vercel에 배포하려고. 순서 알려줘. 환경변수랑 Domain 연결도.
AI
Vercel에서 GitHub 저장소를 연결하면 됩니다. 이후 Push할 때마다 자동으로 Build하고 Deploy합니다.

순서: Vercel 프로젝트 생성 → GitHub 저장소 연결 → 환경변수 입력 → 자동 배포

기본 URL이 생성된 뒤, 직접 구매한 주소는 Domain 설정에서 연결하면 됩니다.
나
배포하기 전에 미리 확인하는 방법 있어?
AI
PR을 열면 Vercel이 Preview URL을 자동 생성합니다. 운영 환경에 합치기 전에 별도 주소로 먼저 확인할 수 있습니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
Vercel
웹사이트와 웹앱을 쉽게 배포할 수 있는 호스팅 플랫폼입니다. GitHub 연결 시 변경마다 자동 배포됩니다.
Build
개발 코드를 실제 배포 가능한 형태로 변환하는 과정입니다. 오류가 있으면 이 단계에서 실패합니다.
Deploy
만든 웹사이트를 인터넷에서 접속할 수 있도록 배포하는 과정입니다.
Domain
사용자가 웹사이트에 접속할 때 입력하는 주소입니다. 배포한 사이트에 읽기 쉬운 주소를 연결합니다.
함께 알아둘 용어
DNS도메인을 실제 서버 주소와 연결해주는 시스템입니다. 도메인 관리 페이지에서 레코드를 추가합니다.
Preview실제 배포 전에 변경 사항을 미리 확인하는 환경입니다. Vercel이 PR마다 preview URL을 자동 생성합니다.
Production실제 사용자가 접속하는 운영 환경입니다. 테스트 환경과 구분해 안정적으로 관리합니다.
STEP 9 / 오류를 만납니다
Error와 Log를 읽어야 고칠 수 있습니다
나
npm run build 돌렸더니 Build Error 났어. Log 보고 고쳐줘. [오류 로그 붙여넣기]
AI
Log를 확인했습니다. Stack Trace 기준으로 components/VideoForm.tsx 34번째 줄에서 타입 불일치로 발생한 Build Error입니다. 해당 줄을 수정하겠습니다. 수정 후 Diff도 함께 보여드리겠습니다.
나
수정됐어. Diff 보니까 다른 곳도 바뀐 것 같은데?
AI
타입 수정하면서 관련 Props 선언도 함께 맞췄습니다. Diff에 표시된 변경 외에 다른 부분은 건드리지 않았습니다.
→ 답변에 나온 단어들, 오른쪽에서 하나씩 풀어봅니다
Error
코드나 설정 문제로 프로그램이 정상 실행되지 않는 상태입니다. 오류 메시지에서 원인 파일을 찾습니다.
Log
프로그램 실행 중 발생한 정보나 오류 기록입니다. 터미널 로그를 AI에게 전달해 원인을 분석합니다.
Build Error
배포 변환 과정에서 발생하는 오류입니다. 타입 오류, 잘못된 import, 환경변수 누락 등이 원인입니다.
Diff
코드가 이전과 비교해 어떻게 바뀌었는지 보여주는 변경 내역입니다. AI 수정 결과를 검토할 때 사용합니다.
함께 알아둘 용어
Type Error데이터의 형태가 예상과 다를 때 발생하는 오류입니다. TypeScript가 미리 잡아줍니다.
Stack Trace오류가 발생한 코드의 호출 흐름을 보여주는 정보입니다. 어느 파일에서 시작됐는지 추적합니다.
STEP 9 보조 / 오류 수정 심화 용어
코드를 더 잘 관리하는 도구들
Runtime Error
앱이 실행되는 중에 발생하는 오류입니다. 페이지는 열리지만 특정 동작에서 실패합니다. API 응답 데이터를 잘못 처리할 때 자주 발생합니다.
Lint
코드 스타일과 잠재적인 문제를 검사하는 과정입니다. 사용하지 않는 변수, 잘못된 문법을 미리 확인합니다. npm run lint로 실행합니다.
Test
코드가 의도한 대로 동작하는지 확인하는 과정입니다. 기능 수정 후 기존 기능이 깨지지 않았는지 검증합니다.
Patch
기존 코드에 적용할 수 있는 변경 묶음입니다. 전체를 다시 작성하지 않고 필요한 부분만 바꿀 때 사용합니다.
Refactor: 기능은 그대로 유지하면서 코드 구조를 더 읽기 쉽고 관리하기 좋게 개선하는 작업입니다. "이 컴포넌트를 더 작은 단위로 나눠서 리팩토링해주세요"처럼 AI에게 요청할 수 있습니다. 기능이 바뀌는 것이 아니므로 Diff로 변경 내역을 반드시 확인하세요.
