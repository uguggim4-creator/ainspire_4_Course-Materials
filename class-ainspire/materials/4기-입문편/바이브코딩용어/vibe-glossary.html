바이브코딩 핵심 용어 사전

앱을 만드는 흐름에 맞춰 자주 만나는 용어 90개를 정리했습니다. 용어를 눌러 펼쳐 보세요.

1.AI에게 작업을 맡길 때
›바이브코딩자연어로 AI와 대화하며 만드는 개발 방식
소개
만들고 싶은 결과를 자연어로 설명하고, AI와 대화하며 코드를 작성하거나 수정하는 개발 방식입니다.
활용 방식
처음부터 모든 코드를 직접 작성하기보다, 목표와 조건을 AI에게 설명하고 결과를 확인하면서 수정해 나갑니다.
활용 예시
"동네 카페 예약 페이지를 만들어주세요. 모바일에서 보기 좋고, 예약 폼이 있어야 합니다."
›에이전트요청을 이해하고 직접 작업을 수행하는 AI
소개
사용자의 요청을 이해하고, 필요한 도구를 사용해 작업을 수행하는 AI입니다.
활용 방식
파일을 읽고, 코드를 수정하고, 테스트를 실행하는 등 실제 개발 작업을 대신 수행할 수 있습니다.
활용 예시
"로그인 페이지를 만들고, 모바일에서도 잘 보이게 수정해주세요"라고 요청하면 에이전트가 관련 파일을 찾아 작업합니다.
›컨텍스트AI가 참고하는 정보 전체
소개
AI가 현재 참고할 수 있는 정보입니다. 대화 내용, 코드 파일, 프로젝트 구조, 사용자의 요청이 포함됩니다.
활용 방식
AI가 정확히 작업하려면 필요한 정보가 컨텍스트에 들어 있어야 합니다. 컨텍스트가 부족하면 AI가 잘못 추측할 수 있습니다.
활용 예시
"기존 디자인과 같은 스타일로 관리자 페이지를 만들어주세요"라고 하면 기존 코드와 대화 내용이 컨텍스트로 사용됩니다.
›프롬프트AI에게 전달하는 작업 지시문
소개
AI에게 전달하는 작업 지시문입니다.
활용 방식
무엇을 만들지, 어떤 조건을 지켜야 하는지, 결과를 어떻게 확인할지 구체적으로 작성하면 좋습니다.
활용 예시
"상품 목록 페이지를 만들어주세요. 카드 형태로 보여주고, 모바일에서는 한 줄에 하나씩 보이게 해주세요."
›서브에이전트특정 역할을 맡는 보조 에이전트
소개
특정 역할을 맡아 작업을 도와주는 보조 에이전트입니다.
활용 방식
큰 작업을 나누어 처리할 때 사용합니다. 한 에이전트는 오류를 분석하고, 다른 에이전트는 UI 구조를 검토하는 식으로 역할을 나눕니다.
활용 예시
코드 리뷰 담당 서브에이전트가 변경된 코드에서 버그 가능성을 검토합니다.
›전역전체 프로젝트에 적용되는 설정
소개
특정 파일이나 함수에만 적용되는 것이 아니라 전체 프로젝트나 넓은 범위에 적용되는 설정을 의미합니다.
활용 방식
전역 스타일, 전역 설정, 전역 변수처럼 여러 곳에서 공통으로 사용하는 값이나 규칙을 관리할 때 사용합니다.
활용 예시
웹사이트 전체의 글꼴과 배경색을 globals.css에서 설정하면 모든 페이지에 동일하게 적용됩니다.
2.프로젝트를 열고 실행할 때
›IDE코드를 작성·수정하는 개발 환경

활용 방식

활용 예시
VS Code나 Cursor에서 프로젝트 폴더를 열고 AI에게 코드 수정을 요청합니다.
›CLI명령어로 프로그램을 사용하는 방식
소개

활용 방식
터미널에서 프로젝트 생성, 패키지 설치, 서버 실행, 배포 명령 등을 실행합니다.
활용 예시
npm run dev 명령어를 입력해 개발 서버를 실행합니다.
›NPMJavaScript 패키지 설치·관리 도구
소개
JavaScript 패키지를 설치하고 관리하는 도구입니다.
활용 방식
필요한 라이브러리를 설치하거나, package.json에 등록된 명령어를 실행할 때 사용합니다.
활용 예시
npm install로 필요한 패키지를 설치하고, npm run dev로 프로젝트를 실행합니다.
›Terminal명령어를 입력해 컴퓨터를 조작하는 창
소개
명령어를 입력해 컴퓨터나 개발 도구를 조작하는 창입니다.
활용 방식
프로젝트 실행, 패키지 설치, 빌드, Git 명령 실행 등에 사용합니다.
활용 예시
터미널에서 npm run dev를 입력해 개발 서버를 실행합니다.
›Bash터미널 명령어를 실행하는 쉘
소개
터미널에서 명령어를 실행할 수 있게 해주는 쉘입니다.
활용 방식
폴더 이동, 파일 확인, 프로그램 실행, 개발 서버 실행 등에 사용합니다.
활용 예시
프로젝트 폴더로 이동한 뒤 npm install을 실행합니다.
›PowerShellWindows에서 자주 쓰는 명령어 실행 환경
소개
Windows에서 자주 사용하는 명령어 실행 환경입니다.
활용 방식
Windows 환경에서 폴더 이동, 파일 확인, 명령 실행, 개발 서버 실행 등에 사용합니다.
활용 예시
PowerShell에서 npm run dev를 입력해 개발 서버를 실행합니다.
›Read파일이나 데이터를 읽는 동작
소개
파일이나 데이터를 읽는 동작을 의미합니다.
활용 방식
AI가 기존 코드 구조를 파악하거나 사용자가 파일 내용을 확인할 때 사용됩니다.
활용 예시
AI에게 "현재 로그인 페이지 코드를 읽고 구조를 설명해주세요"라고 요청합니다.
›Write파일이나 데이터에 내용을 쓰는 동작
소개
파일이나 데이터에 내용을 쓰는 동작을 의미합니다.
활용 방식
새 파일을 만들거나 기존 파일을 수정할 때 사용됩니다.
활용 예시
AI에게 "회원가입 페이지를 만들어주세요"라고 요청하면 필요한 파일에 코드를 작성합니다.
›package.json프로젝트 설정과 패키지 정보를 담는 파일
소개

활용 방식
프로젝트 이름, 실행 명령어, 설치된 패키지 목록을 관리합니다.
활용 예시
scripts 항목에 있는 dev, build, start 명령어를 NPM으로 실행합니다.
›node_modules설치된 패키지가 저장되는 폴더
소개

활용 방식
npm install을 실행하면 package.json에 적힌 패키지들이 이 폴더에 설치됩니다.
활용 예시
프로젝트를 처음 받은 뒤 npm install을 실행하면 node_modules 폴더가 생성됩니다.
›Localhost내 컴퓨터의 개발 서버 주소
소개

활용 예시

개발 서버가 어떤 주소에서 실행되는지 구분할 때 사용합니다.
활용 예시
localhost:3000에서 3000이 port 번호입니다.
3.화면을 만들 때
›HTML웹페이지의 구조를 만드는 언어
소개
웹페이지의 구조를 만드는 언어입니다.
활용 방식
제목, 문단, 버튼, 이미지, 입력창 같은 화면 요소의 기본 구조를 만듭니다.
활용 예시
상품 이름은 제목으로, 상품 설명은 문단으로, 구매 버튼은 버튼 요소로 구성합니다.
›CSS웹페이지의 디자인을 담당하는 언어
소개
웹페이지의 디자인을 담당하는 언어입니다.
활용 방식
색상, 글꼴, 여백, 배치, 반응형 화면 등을 설정합니다.
활용 예시
버튼의 배경색을 바꾸고, 모바일 화면에서는 카드가 한 줄에 하나씩 보이게 만듭니다.
›JavaScript웹페이지에 동작을 더하는 언어
소개
웹페이지에 동작을 추가하는 프로그래밍 언어입니다.
활용 방식
버튼 클릭, 데이터 요청, 화면 변경 같은 기능을 구현할 때 사용합니다.
활용 예시
사용자가 버튼을 클릭하면 메뉴가 열리도록 만듭니다.
›React화면을 컴포넌트로 만드는 라이브러리
소개
화면을 컴포넌트 단위로 만들 수 있게 해주는 JavaScript 라이브러리입니다.
활용 방식
버튼, 카드, 입력창 같은 화면 요소를 재사용 가능한 컴포넌트로 나눕니다.
활용 예시
ProductCard 컴포넌트를 만들어 여러 상품을 같은 디자인으로 보여줍니다.
›Next.jsReact 기반 웹앱 프레임워크
소개
React 기반 웹앱 프레임워크입니다. 페이지 구성, 라우팅, 서버 기능, 배포 최적화 등을 제공합니다.
활용 방식
React로 웹앱을 만들 때 필요한 기본 구조를 제공해 더 빠르게 개발할 수 있습니다.
활용 예시
블로그, 쇼핑몰, 관리자 페이지 같은 웹앱을 Next.js로 만듭니다.

소개
JavaScript에 타입 개념을 추가한 언어입니다.

Header, Button, LoginForm 같은 컴포넌트를 만들어 사용합니다.
›Props부모가 자식 컴포넌트에 전달하는 데이터
소개

/login 주소로 접속하면 로그인 페이지, /dashboard로 접속하면 관리자 페이지를 보여줍니다.
