# 교안(슬라이드 deck) 제작 매뉴얼

AInspire 자료집 포털에 들어가는 **HTML 슬라이드형 교안**을 만드는 표준 방법입니다.
"AI가 활용하는 미디어 도구 지도"(`materials/미디어도구/media-tools-deck.html`)가 기준 템플릿입니다.

---

## 0. 전체 그림

```
class-ainspire/
├─ index.html                      # 포털 메인 (카드 그리드 + 모달)
├─ data-materials.js               # 자료 목록 데이터 (카드 한 장 = 객체 하나)
├─ viewer.html                     # 'article' 타입 본문 뷰어 (모달에 임베드됨)
└─ materials/
   └─ <자료폴더>/
      ├─ <deck>.html               # 슬라이드 교안 본체
      └─ demo-assets/              # 교안에 쓰는 영상·이미지
```

교안을 새로 만들면 **두 가지**만 하면 포털에 뜹니다.
1. `materials/<폴더>/<deck>.html` 슬라이드 파일 작성
2. `data-materials.js`에 카드 객체 1개 추가

## 1. 디자인 토큰 (항상 고정)

모든 교안은 이 값으로 통일합니다. 색은 **2~3개만**, 폰트는 **Paperlogy 하나**.

```css
:root {
  --ink:    #1a1530;   /* 본문 텍스트 (진한 남보라) */
  --purple: #6020ff;   /* 포인트 (제목 accent, 라벨) */
  --line:   #e5dff0;   /* 테두리·구분선 */
  --bg:     #ffffff;   /* 배경 (교안은 흰 바탕) */
  --purple-light: #f7f3ff;  /* 카드 배경 */
}
```

**Paperlogy 폰트 (CDN, @font-face 3종):**
```css
@font-face{font-family:'Paperlogy';src:url('https://cdn.jsdelivr.net/gh/projectnoonnu/2408-3@1.0/Paperlogy-4Regular.woff2') format('woff2');font-weight:400;font-display:swap;}
@font-face{font-family:'Paperlogy';src:url('https://cdn.jsdelivr.net/gh/projectnoonnu/2408-3@1.0/Paperlogy-7Bold.woff2') format('woff2');font-weight:700;font-display:swap;}
@font-face{font-family:'Paperlogy';src:url('https://cdn.jsdelivr.net/gh/projectnoonnu/2408-3@1.0/Paperlogy-8ExtraBold.woff2') format('woff2');font-weight:800;font-display:swap;}
```

> ffmpeg 자막 등에서 Paperlogy를 지정할 때 내부 패밀리명은 `Paperlogy 7 Bold` (그냥 `Paperlogy` 아님).

## 2. 콘텐츠 원칙 (말투·구성)

- **존댓말**로 작성. 일상 비유 금지.
- 이모지 금지.
- 도구 슬라이드는 항상 **3단 구성**:
  1. **도구 소개** — 이 도구가 뭔지 (1~2문장)
  2. **활용 방식** — 제작 파이프라인에서 어디에 쓰는지
  3. **활용 예시** — *명령어가 아니라* "AI에게 이렇게 요청하세요" + "이런 결과를 받습니다"
- 요청 예시 문장에는 **도구 이름을 넣는다.**
  예) "**yt-dlp를 활용해서** 이 유튜브 영상 원본이랑 음성만 따로 받아줘."
- 우측에는 **실제 실행/결과 영상**을 넣는다 (캡처가 아니라 진짜 돌아가는 장면).

## 3. 슬라이드 deck 구조

### 3-1. 뼈대

```html
<body>
  <div class="deck">
    <!-- 슬라이드들 -->
    <div class="slide centered" data-index="0"> ... 타이틀 ... </div>
    <div class="slide" data-index="1"> ... 도구 슬라이드 ... </div>
    ...
  </div>
  <div class="progress"><div id="progressBar"></div></div>
  <div id="pageIndicator">1 / N</div>
  <script> ... 네비게이션 + 임베드 fit 로직 ... </script>
</body>
```

- `.slide` = 한 장 (절대위치, `inset:0`, flex column).
- `.slide.centered` = 타이틀/마무리처럼 **가운데 정렬**할 슬라이드.
- `.slide.active` 가 현재 보이는 장. JS가 `translateX`로 전환.
- 슬라이드 개수는 JS 상단 `const TOTAL = N;` 과 일치시킬 것.

### 3-2. 도구 슬라이드 마크업 (복붙용 템플릿)

```html
<div class="slide" data-index="N">
  <div class="step-label">Step N / 도구이름</div>
  <div class="slide-title"><span class="accent">도구이름</span> — 한 줄 요약</div>
  <div class="divider"></div>

  <div class="tool-body">
    <!-- 좌: 본문 -->
    <div class="tool-left">
      <div class="info-block">
        <div class="info-block-label">도구 소개</div>
        <div class="info-block-text">…</div>
      </div>
      <div class="info-block">
        <div class="info-block-label">활용 방식</div>
        <div class="info-block-text">…</div>
      </div>
      <div class="example-block">
        <div>
          <div class="ex-label">이렇게 요청하세요</div>
          <div class="ex-request"><div class="ex-request-text">"…요청 문장…"</div></div>
        </div>
        <div>
          <div class="ex-label">이런 결과를 받습니다</div>
          <div class="ex-result"><div class="ex-result-text">…</div></div>
        </div>
      </div>
    </div>

    <!-- 우: 실제 영상 (영상1=대화+터미널 흐름 / 영상2=결과물) -->
    <div class="tool-right">
      <div class="demo-split">
        <div class="demo-pair">
          <div class="demo-pair-label">1. 요청 → 실행</div>
          <video class="demo-clip" src="demo-assets/rec-도구-flow.mp4" autoplay loop muted playsinline></video>
        </div>
        <div class="demo-pair">
          <div class="demo-pair-label">2. 결과물 — …</div>
          <video class="demo-clip result" src="demo-assets/결과.mp4" autoplay loop muted playsinline></video>
          <div class="demo-credit">출처 표기 (저작권 영상이면 필수)</div>
        </div>
      </div>
    </div>
  </div>
</div>
```

핵심 클래스 의미:
| 클래스 | 역할 |
|---|---|
| `.tool-body` | 좌우 2단 그리드 (`grid-template-columns: 1fr 1.05fr`) |
| `.tool-left` / `.tool-right` | 좌측 본문 / 우측 영상 컬럼 |
| `.demo-clip` | 16:9 영상. `.result`를 붙이면 `object-fit: contain` (세로 영상 레터박스) |
| `.demo-credit` | 출처 캡션 (CC 영상 등은 반드시) |

## 4. 모달 임베드 대응 (★ 이번에 핵심으로 잡은 부분)

포털에서 카드를 누르면 deck이 **16:9 모달 안 iframe**으로 열립니다 (`?embed=1`).
풀스크린용 슬라이드를 16:9 박스에 욱여넣으면 넘치거나 여백이 생기므로, 아래 2가지가 들어가야 합니다.

### 4-1. 임베드 감지 (deck JS 최상단)

```js
if (new URLSearchParams(location.search).get('embed') === '1') {
  document.body.classList.add('embed-mode');
}
```

### 4-2. 임베드 전용 CSS (deck `</style>` 직전)

```css
/* 도구 슬라이드: 제목 상단 고정, 본문 영역이 남은 높이를 채움.
   타이틀/마무리(.centered)는 기존 중앙 정렬 유지 */
body.embed-mode .slide:not(.centered) {
  justify-content: flex-start;
  overflow: hidden;
}
body.embed-mode .slide:not(.centered) .step-label,
body.embed-mode .slide:not(.centered) .slide-title,
body.embed-mode .slide:not(.centered) .slide-subtitle,
body.embed-mode .slide:not(.centered) .divider { flex-shrink: 0; }

/* 제목-본문 붙이기 */
body.embed-mode .slide:not(.centered) .slide-title { margin-bottom: clamp(4px, 0.8vh, 8px); }
body.embed-mode .slide:not(.centered) .divider { margin-bottom: clamp(12px, 2vh, 22px); }

/* 본문 영역: 좌우 컬럼 같은 높이로, 상단부터 정렬 */
body.embed-mode .tool-body {
  flex: 1;
  min-height: 0;
  align-items: stretch;
  grid-template-columns: 1.15fr 1fr;   /* 좌측 본문 넓혀 영상을 더 오른쪽으로 */
  gap: clamp(64px, 7vw, 120px);        /* 좌측 본문 ↔ 우측 영상 간격 */
}
body.embed-mode .tool-left  { justify-content: flex-start; }
body.embed-mode .tool-right { justify-content: flex-start; }

/* 본문 글자 키우기 */
body.embed-mode .info-block-text { font-size: clamp(15px, 1.4vw, 19px); }
body.embed-mode .ex-request-text { font-size: clamp(15px, 1.4vw, 19px); }
body.embed-mode .ex-result-text  { font-size: clamp(14px, 1.3vw, 18px); }
body.embed-mode .info-block-label,
body.embed-mode .ex-label { font-size: clamp(11px, 1vw, 13px); }
```

### 4-3. 자동 맞춤(zoom) 로직 (deck JS)

콘텐츠가 16:9 박스보다 높으면 슬라이드를 통째로 `zoom` 축소합니다.
(transform 전환 애니메이션과 충돌 안 하도록 `transform: scale`이 아니라 `zoom` 사용)

```js
const IS_EMBED = document.body.classList.contains('embed-mode');

function fitSlide(slide) {
  if (!IS_EMBED || !slide) return;
  slide.style.zoom = '1';
  requestAnimationFrame(() => {
    const cs = getComputedStyle(slide);
    const padV = parseFloat(cs.paddingTop) + parseFloat(cs.paddingBottom);
    const gap  = parseFloat(cs.rowGap) || 0;
    let needed = padV;
    Array.from(slide.children).forEach((child, i) => {
      if (i > 0) needed += gap;
      if (child.classList.contains('tool-body')) {
        // grid 자식(tool-left/right) 중 가장 높은 콘텐츠
        let inner = 0;
        for (const col of child.children) inner = Math.max(inner, col.getBoundingClientRect().height);
        needed += inner;
      } else {
        needed += child.getBoundingClientRect().height;
      }
    });
    const avail = slide.clientHeight;
    slide.style.zoom = needed > avail + 1 ? Math.max(0.5, avail / needed).toFixed(3) : '1';
  });
}
```

호출 위치:
```js
// goTo(index) 끝에서
fitSlide(slides[current]);

// 초기화 + 리사이즈 + 미디어 로드 후 재측정
fitSlide(slides[0]);
window.addEventListener('resize', () => fitSlide(slides[current]));
if (IS_EMBED) {
  document.querySelectorAll('video, img').forEach(el => {
    el.addEventListener('loadeddata', () => fitSlide(slides[current]));
    el.addEventListener('load',       () => fitSlide(slides[current]));
  });
  window.addEventListener('load', () => fitSlide(slides[current]));
}
```

> **튜닝 포인트**
> - 영상을 더 오른쪽으로 → `grid-template-columns` 좌측 비율↑ 또는 `gap`↑
> - 본문 더 크게 → `font-size` clamp 상한↑ (단 너무 키우면 zoom 축소가 먹혀 역효과)
> - 슬라이드가 잘림 → zoom 하한(`0.5`)을 낮추거나 콘텐츠를 줄임

## 5. data-materials.js 에 등록

카드 한 장 = 객체 한 개. **배열 맨 앞에 넣으면 포털 1번**으로 노출됩니다.

```js
{
  id: "고유-id",                  // 영문 kebab-case, 중복 금지
  category: "common",             // week1 | week2 | common | cineguide
  categoryKo: "미디어 도구",
  title: "교안 제목",
  summary: "한 줄 설명…",
  fileType: "link",               // ↓ 아래 표 참고
  filePath: "materials/<폴더>/<deck>.html",
  driveId: "구글드라이브파일ID",    // pdf/excel일 때만
  readTime: "슬라이드 · N장",
  date: "표시용 라벨",
  icon: "clapperboard"            // lucide 아이콘명
}
```

### fileType 별 모달 동작
| fileType | 모달에서 뜨는 방식 | 비고 |
|---|---|---|
| `link` | deck/CineGuide HTML을 `?embed=1` iframe | **슬라이드 교안은 이걸 사용** |
| `article` | `viewer.html?id=…&embed=1` iframe | 로컬 HTML 본문 (헤더/푸터 숨김) |
| `pdf` / `excel` | `drive.google.com/file/d/<driveId>/preview` | **driveId 필수**, 파일은 공개 폴더에 |

### PDF·Excel을 모달로 띄우려면 (Google Drive)
1. 파일을 **공개 폴더**(현재: `AInspire 4기 수강생 자료 (공개)`, id `1XD7fvn9UoLc6WyKxlZBZvXjP6k519cwr`)에 복사
2. 폴더가 "링크가 있는 모든 사용자"면 복사본도 공개 상속됨
3. 파일 ID를 `driveId`에 기입
> Drive 도구로는 **권한 변경이 불가** → 폴더를 1회 공개로 설정해 두고 그 안에 복사하는 방식.

## 6. 실제 실행/결과 영상 만들기

캡처가 어려우면 **실행 로그로 터미널풍 영상**을 ffmpeg `drawtext`로 생성합니다.

- 영상1 (`rec-도구-flow.mp4`): "AI에게 대화로 요청 → 터미널 실행" 흐름
- 영상2 (결과물): 실제 산출물 영상 (세로 결과는 `.demo-clip.result`로 레터박스)
- ffmpeg drawtext 주의:
  - `%` 들어가면 `expansion=none`
  - 한글은 `malgun.ttf` (consola.ttf는 □□로 깨짐)
- 출처 있는 영상(CC 등)은 `.demo-credit`에 **반드시 표기**

## 8. 로컬 미리보기

```bash
cd class-ainspire
python -m http.server 8765
# http://127.0.0.1:8765/index.html
```
`fetch`(article 본문 로드)가 `file://`에선 막히므로 **반드시 HTTP 서버**로 띄울 것.
