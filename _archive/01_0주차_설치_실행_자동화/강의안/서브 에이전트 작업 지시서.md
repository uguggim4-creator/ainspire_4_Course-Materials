# 서브 에이전트 작업 지시서: ainspire 4기 0주차 강의안 세부 내용 작성

## 1. 작업 목적

이 문서는 0주차 강의 목차를 여러 서브 에이전트가 나누어 채우기 위한 공통 작업 지시서입니다.
각 서브 에이전트는 자신에게 배정된 목차만 세부 작성합니다.
최종 결과물은 이후 PPT 제작의 원천 자료로 사용됩니다.

---

## 2. 기준 문서

세부 내용을 채울 기준 목차 문서:

```text
E:\ainspire 4기 강의\01_0주차_설치_실행_자동화\강의안\0주차 강의내용 정리.md
```

서브 에이전트는 위 문서의 목차 번호와 제목을 유지하면서 내용을 작성합니다.

---

## 3. 전체 작성 원칙

### 3.1 대상 독자

- 개발 경험이 많지 않은 비개발자
- AI 도구를 실무/콘텐츠/문서/강의 제작에 활용하려는 사람
- CLI, API, IDE 같은 개발 용어가 낯선 사람
- AI 도구를 설치하고 실제 파일 결과물을 만들고 싶은 사람

### 3.2 설명 톤

- 개발자 문서처럼 쓰지 않는다.
- 개념보다 용도와 상황을 먼저 설명한다.
- 어려운 용어는 반드시 쉬운 비유 또는 예시를 붙인다.
- 과장된 마케팅 표현을 피한다.
- 특정 도구를 무조건 추천하지 않고, 상황별 선택 기준을 제시한다.

### 3.3 문장 규칙

- 한 문장은 가능하면 짧게 쓴다.
- 한 문단은 3~5문장 이내로 유지한다.
- 영어 용어는 처음 등장할 때 한국어 설명을 붙인다.
- 수강생이 따라 할 수 있는 표현을 사용한다.
- PPT로 옮기기 쉬운 핵심 문장을 따로 뽑는다.

### 3.4 금지 사항

- 확인되지 않은 최신 정보 단정 금지
- 모델 성능 순위 단정 금지
- 가격, 요금, 컨텍스트 길이 같은 최신 정보의 출처 없는 작성 금지
- API 키, 토큰, 개인정보 예시를 실제 값처럼 작성 금지
- 특정 서비스의 비공식 설치 파일 안내 금지
- 저작권이 있는 유튜브/이미지 자료의 무단 사용 권장 금지

## 4. 각 목차별 기본 산출 형식

각 서브 에이전트는 맡은 목차를 아래 형식으로 작성합니다.

```markdown
## [목차 번호]. [목차 제목]

### 핵심 메시지
- 이 목차에서 수강생이 반드시 이해해야 할 한 문장

### 설명
본문 설명을 작성합니다.

### 비개발자식 설명
어려운 개념을 쉬운 말로 다시 설명합니다.

### 예시
수강생이 바로 이해할 수 있는 예시를 작성합니다.

### 실습 또는 확인 질문
- 수강생이 직접 해볼 수 있는 행동
- 또는 이해 확인 질문

### PPT 반영 포인트
- 슬라이드 제목 후보
- 슬라이드에 넣을 핵심 bullet
- 표/도식/캡쳐가 필요한지

### 캡쳐/이미지 플레이스홀더
- [이미지 자리] 필요한 화면 설명

### 참고/출처
- 공식 문서 또는 신뢰 가능한 출처 링크
- 최신 정보가 필요한 경우 확인 날짜를 함께 표기
```

## 5. 분량 기준

### 짧은 개념 목차
- 본문 400~800자
- 예시 1~2개
- PPT 반영 포인트 3~5개

### 설치/실행 가이드 목차
- 본문 800~1,500자
- 단계별 체크리스트 포함
- 캡쳐 플레이스홀더 포함
- 자주 생기는 문제 2~3개 포함

### 비교/선택 기준 목차
- 본문 800~1,500자
- 비교표 포함
- 상황별 추천 포함
- 단정 대신 판단 기준 중심 작성

### 보안/승인/요금 목차
- 본문 800~1,500자
- 주의사항 체크리스트 포함
- 위험한 예시와 안전한 예시 구분
- 공식 문서 출처 우선

## 10. 메인 에이전트 통합 방식

서브 에이전트들이 작성한 세부 내용은 이후 메인 에이전트가 통합합니다.

- 중복 설명 제거
- 용어 표현 통일
- 슬라이드 흐름 재배치
- 너무 긴 본문 압축
- 표와 도식 우선 배치
- 캡쳐 플레이스홀더를 실제 이미지로 교체
- 최종 PPT 디자인 적용

최종 PPT 제작 방향:

- 실습 파트는 체크리스트와 결과물 예시 중심으로 만든다.
