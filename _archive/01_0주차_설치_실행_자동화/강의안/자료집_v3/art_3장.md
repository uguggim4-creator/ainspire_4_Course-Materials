## 3. AI로 다른 결과를 내는 핵심 — 도구 활용

챗봇만 사용할 때 AI는 주로 “말로 답하는 도구”에 가깝습니다. 하지만 로컬 AI에 도구를 연결하면, 권한을 부여받은 범위 안에서 자료를 찾고, 파일을 만들고, 브라우저를 조작하고, 결과물을 정리하는 일까지 이어서 처리할 수 있습니다. **차이는 AI가 말을 얼마나 잘하느냐보다, 어떤 도구를 안전하게 쓸 수 있느냐에서 크게 납니다.**

영상 크리에이터에게 중요한 것은 도구 이름을 많이 외우는 일이 아닙니다. 각 도구가 어느 부류에 속하는지 이해하면, 작업 흐름을 훨씬 쉽게 설계할 수 있습니다:

- 자료 수집·웹 자동화 도구
- 이미지·영상 생성 모델과 서비스
- 음성·음악 도구
- 영상 후반·조립 도구
- 문서·디자인·협업 도구
- 결과물 확인·관리 도구

로컬 AI라고 해서 모든 작업이 반드시 내 컴퓨터 안에서만 끝나는 것은 아닙니다. Replicate, CometAPI, Midjourney, Suno 같은 외부 서비스나 API를 쓰면 프롬프트, 이미지, 영상, 음성 자료가 해당 서비스로 전송될 수 있습니다. **외부 도구를 연결할 때는 2026-05 기준 공식 문서와 약관 확인이 필요합니다.**

## 3.1 자료 수집·웹 자동화

자료 수집·웹 자동화 도구는 인터넷에서 자료를 가져오거나, 웹페이지를 열고, 반복적인 클릭·스크롤·요청 작업을 대신하게 하는 부류입니다. 로컬 AI가 권한을 부여받은 경우, 레퍼런스 수집, 자막 확보, 링크 정리, 메타데이터 확인 같은 일을 자동화할 수 있습니다.

### yt-dlp

yt-dlp는 영상·자막 등을 내려받는 명령 도구(CLI: 글자로 명령을 입력해 실행하는 방식)입니다. 여러 영상 사이트에서 영상, 오디오, 자막, 메타데이터를 내려받는 데 쓰이며, 로컬 AI가 이 명령을 실행할 권한을 부여받은 경우 채널 백업, 자막 추출, 자료 정리 흐름에 포함될 수 있습니다. **강력한 다운로드 기능은 권리와 허용 범위가 분명할 때만 안전하게 쓸 수 있습니다.**

쉽게 비유하면, 합법적으로 받을 수 있는 영상 자료를 대신 받아 주는 “다운로드 전문 직원”에 가깝습니다. 다만 아무 자료나 마음대로 가져와도 된다는 뜻은 아닙니다. 본인이 권리를 가진 자료이거나 사용이 허용된 자료에서만 사용해야 하며, 사이트 약관과 저작권을 함께 확인해야 합니다.

영상 크리에이터는 본인 채널 영상 백업, 허가받은 자료의 자막 추출, 강의·인터뷰 영상의 스크립트 정리, 레퍼런스 영상 목록 관리에서 yt-dlp를 접하게 됩니다. yt-dlp가 “영상 사이트에서 자료를 찾아 내려받는 도구”라면, ffmpeg는 내려받은 영상·오디오를 자르거나 변환하는 “후반 처리 도구”입니다. Playwright는 웹페이지를 사람이 보는 것처럼 조작하는 도구라서 역할이 다릅니다.

### Playwright

Playwright는 브라우저를 자동으로 조작해 클릭, 입력, 스크롤, 페이지 이동을 코드로 대신하게 하는 도구입니다. 사람이 웹브라우저에서 하는 행동을 코드로 재현하고, 페이지가 불러오는 네트워크 요청도 관찰할 수 있습니다. **웹사이트를 실제 브라우저 화면 기준으로 다뤄야 할 때 유용합니다.**

권한이 있거나 허용된 범위에서는 영상 사이트의 목록을 정리하거나, 페이지 안에서 실제로 불러오는 영상 URL 후보를 탐지하는 데 쓰일 수 있습니다. 마우스와 키보드를 대신 움직여 주는 “웹브라우저 조작 로봇 손”이라고 보면 이해하기 쉽습니다.

영상 플랫폼의 업로드 상태 확인, 레퍼런스 목록 수집, 썸네일 후보 페이지 정리, 반복적인 웹서비스 조작 자동화에서 자주 등장합니다. curl이 브라우저를 보지 않고 서버에 직접 요청을 보내는 도구라면, Playwright는 실제 브라우저를 띄워 자바스크립트가 실행된 화면까지 다룰 수 있습니다. API가 정해진 통로로 데이터를 주고받는 방식이라면, Playwright는 사람이 웹사이트를 쓰는 행동을 자동화하는 방식입니다.

### curl

curl은 인터넷 주소로 데이터를 요청하거나 보내는 기본 명령 도구입니다. API(Application Programming Interface: 프로그램끼리 정해진 형식으로 요청·응답하는 통로)를 호출할 때 자주 쓰이며, 브라우저를 열지 않고도 특정 주소에 요청을 보내고 응답을 확인하거나 파일을 내려받을 수 있습니다. **서버에 직접 “요청 편지”를 보내는 도구라고 생각하면 쉽습니다.**

Replicate, CometAPI, Notion 같은 서비스의 API가 잘 작동하는지 확인할 때도 curl이 자주 등장합니다. AI 모델 API 호출 테스트, 이미지 생성 요청, Notion 데이터베이스 업데이트, 파일 업로드·다운로드, 자동화 스크립트 점검에 쓰입니다.

curl을 사용할 때 등장하는 인증 토큰 또는 API 키는 비밀번호처럼 권한을 증명하는 값입니다. 이는 AI 토큰, 즉 모델이 글자를 처리할 때 계산하는 단위와는 다릅니다. 또한 curl은 브라우저 자동화 도구가 아니므로, 클릭이나 스크롤이 필요한 작업에는 Playwright가 더 적합합니다.

## 3.2 이미지·영상 생성 (생성 AI)

이미지·영상 생성 도구는 프롬프트, 참고 이미지, 설정값을 바탕으로 새로운 시각 결과물을 만드는 부류입니다. 이 영역에서는 “모델”, “서비스”, “플랫폼”, “MCP 연결 방식”이 서로 다른 층위라는 점을 먼저 이해하는 것이 좋습니다. **같은 생성 AI처럼 보여도, 실제로 결과물을 만드는 주체와 그 주체를 호출하는 통로는 다를 수 있습니다.**

### Higgsfield(MCP)

Higgsfield는 이미지나 영상을 생성하는 서비스입니다. 반면 MCP(Model Context Protocol: AI 앱이 외부 도구와 자료에 연결되는 약속)는 생성 모델이 아니라, AI 앱과 외부 서비스 사이를 이어 주는 연결 방식입니다. **Higgsfield가 영상을 만들어 주는 공장이라면, MCP는 그 공장에 주문서를 전달하는 연결 통로에 가깝습니다.**

Higgsfield가 MCP로 연동되는지, 공식 지원인지, 커뮤니티 서버인지, 어떤 기능을 호출할 수 있는지는 2026-05 기준 공식 문서 확인이 필요합니다. MCP 연동이라는 말이 보이더라도 그것이 곧 Higgsfield 자체를 뜻하는 것은 아닙니다.

영상 작업에서는 쇼츠용 B-roll 생성, 움직이는 장면 후보 제작, 콘셉트 영상 테스트, 이미지에서 짧은 영상으로 확장하는 작업에서 접할 수 있습니다. 생성은 Higgsfield 서비스가 하고, MCP는 AI 앱이 그 서비스를 부를 수 있게 하는 연결 방식입니다. Replicate나 CometAPI처럼 여러 모델을 호출하는 플랫폼·게이트웨이와도 층위가 다릅니다.

### Seedance

Seedance는 텍스트나 참고 이미지를 바탕으로 영상을 생성하는 영상 생성 모델 계열입니다. 특정 영상 생성 모델 또는 모델 계열 이름으로 접하게 될 수 있으며, 생성 가능한 영상 길이, 해상도, 입력 방식, 제공 플랫폼, API 지원 여부는 제공 경로에 따라 달라질 수 있습니다. **영상 편집기라기보다는 새로운 영상 컷을 만들어 내는 생성 모델로 이해하는 편이 좋습니다.**

대본 속 한 장면을 짧은 움직이는 컷으로 뽑아 주는 “영상 생성 담당자”에 가깝습니다. 쇼츠의 삽입 컷, 배경 움직임, 광고 영상의 콘셉트 장면, 실제 촬영 전 시각화용 영상 후보를 만들 때 쓰일 수 있습니다.

Seedance는 모델 계열이고, Replicate나 CometAPI는 모델을 호출하게 해주는 플랫폼 또는 게이트웨이입니다. ffmpeg나 Remotion은 생성된 영상을 자르거나 조립하는 후반 도구이므로, Seedance와 역할이 다릅니다.

### GPT-Image-2

GPT-Image-2는 텍스트나 참고 이미지를 바탕으로 이미지를 생성·편집하는 GPT 계열 이미지 생성 모델명으로 접할 수 있는 항목입니다. 실제 공식 모델명, 버전, 제공 방식, 가격, 입력 가능한 이미지 조건은 변동될 수 있으므로 2026-05 기준 공식 문서 확인이 필요합니다. **이름이 모델을 가리키더라도, 실제 사용 조건은 제공 방식에 따라 달라질 수 있습니다.**

보통 이런 이미지 모델은 API나 통합 서비스, 또는 AI 앱의 도구 연결을 통해 호출됩니다. 글로 설명한 썸네일 시안이나 장면 이미지를 그려 주는 “이미지 제작 담당자”처럼 생각하면 쉽습니다.

유튜브 썸네일 시안, 스토리보드, 장면 콘셉트 아트, 제품 이미지 변형, 배경 이미지 제작에 쓰일 수 있습니다. GPT-Image-2는 모델명이고, ChatGPT 같은 앱은 모델을 사용하는 제품 인터페이스입니다. CometAPI 같은 통합 API를 통해 호출하는 경우에도, CometAPI는 모델 자체가 아니라 연결 창구입니다.

### Midjourney(MJ)

Midjourney(MJ)는 프롬프트를 바탕으로 이미지를 생성하는 이미지 생성 모델·서비스 계열입니다. 강한 스타일감과 시각적 완성도로 많이 알려져 있으며, 분위기, 색감, 구도, 스타일을 말로 설명하면 시각 시안을 만들어 주는 “AI 미술 감독”에 가깝습니다. **시각적 콘셉트와 무드보드를 빠르게 잡을 때 자주 언급되는 도구입니다.**

접근 방식은 웹, 앱, 커뮤니티 기반 인터페이스, API 또는 외부 통합 방식 등으로 바뀔 수 있습니다. 자동화 사용, 상업적 이용, 이미지 권리 조건도 요금제와 정책에 따라 달라질 수 있으므로 2026-05 기준 공식 문서 확인이 필요합니다.

영상 크리에이터는 썸네일 배경, 인물·캐릭터 콘셉트, 채널 브랜딩 이미지, 포스터형 이미지, 영상 분위기 참고용 무드보드 제작에서 Midjourney를 접할 수 있습니다. Midjourney는 이미지 생성 모델·서비스이고, CometAPI는 여러 모델을 한 형식으로 호출하게 해주는 게이트웨이입니다. MJ를 어떤 게이트웨이를 통해 쓰더라도, 원 서비스의 정책과 게이트웨이의 정책은 따로 확인해야 합니다.

### nano-banana

nano-banana는 이미지 생성 모델 또는 별칭으로 접할 수 있는 이름이며, 정확한 공식 명칭과 제공 경로 확인이 필요한 항목입니다. 모델 목록이나 통합 서비스에서 이미지 생성 모델 이름처럼 등장할 수 있지만, 공식 모델명인지, 별칭인지, 특정 제공사의 내부 이름인지, 게이트웨이에서 붙인 표시명인지는 확인이 필요합니다. **이름만 보고 같은 모델이라고 단정하기 어렵습니다.**

같은 이름처럼 보여도 실제 성능, 입력 방식, 가격, 권한 정책이 제공처마다 다를 수 있습니다. 메뉴판에 적힌 이미지 모델 이름은 맞지만, 어느 주방에서 어떤 방식으로 나오는 메뉴인지 확인해야 하는 항목이라고 보면 됩니다.

썸네일 변형, 참고 이미지 기반 재구성, 캐릭터 이미지 테스트, 콘셉트 이미지 생성 등에서 모델 선택지로 볼 수 있습니다. GPT-Image 계열, Midjourney, 다른 이미지 모델과 비교할 때는 이름만 보지 말고 제공사, 입력 방식, 사용권, 결과물 권리를 함께 확인해야 합니다.

### Replicate

Replicate는 여러 AI 모델을 한곳에서 API로 호출할 수 있게 해주는 모델 실행 플랫폼입니다. 이미지, 영상, 음성, 업스케일, 분류 등 다양한 AI 모델을 API로 실행할 수 있게 해주며, 모델 자체라기보다는 여러 모델을 올려 두고 실행할 수 있는 공간에 가깝습니다. **여러 AI 도구가 입점해 있는 공용 작업장이라고 생각하면 이해하기 쉽습니다.**

필요한 모델을 골라 작업을 맡기는 방식이며, 모델별 버전, 속도, 비용, 라이선스는 각각 다를 수 있습니다. 그래서 Replicate를 쓴다고 해서 모든 모델의 조건이 같아지는 것은 아닙니다.

이미지 생성, 영상 생성, 이미지 확대, 배경 제거, 음성 처리, 실험용 모델 비교, 자동화 파이프라인 제작에 쓰입니다. Replicate가 플랫폼 안에서 제공되는 모델을 실행하는 구조라면, CometAPI는 여러 외부 모델·서비스를 하나의 API로 묶는 게이트웨이에 가깝습니다.

### CometAPI

CometAPI는 여러 생성 모델과 서비스를 하나의 API 형식으로 호출하게 해주는 통합 게이트웨이입니다. Midjourney, Suno, GPT-Image 계열 등 여러 모델을 하나의 API 방식으로 호출하도록 지원하는 게이트웨이로 소개되는 경우가 있습니다. **CometAPI 자체가 생성 모델은 아니며, 여러 모델로 요청을 보내는 연결 창구에 가깝습니다.**

어떤 모델을 실제로 지원하는지, 원 서비스와 기능 차이가 있는지, 가격과 속도, 권리 조건이 어떻게 다른지는 2026-05 기준 공식 문서 확인이 필요합니다. 여러 식당에 각각 전화하지 않고, 한 배달 창구에 주문을 넣는 방식으로 비유할 수 있습니다.

로컬 AI가 썸네일은 이미지 모델로, BGM은 음악 모델로, 영상 컷은 영상 모델로 나누어 호출하는 자동화 흐름에서 쓰일 수 있습니다. CometAPI는 게이트웨이이고, API는 프로그램끼리 요청을 주고받는 통로이며, MCP는 AI 앱이 도구를 사용할 수 있게 연결하는 약속입니다. CometAPI를 MCP 서버가 감싸서 제공할 수는 있지만, API와 MCP는 같은 개념이 아닙니다.

## 3.3 음성·음악

음성·음악 도구는 대본을 목소리로 바꾸거나, 분위기에 맞는 음악을 생성하는 부류입니다. 영상 크리에이터에게는 품질만큼이나 목소리 권리, 상업적 이용 조건, 플랫폼 업로드 정책 확인이 중요합니다.

### ElevenLabs(TTS)

ElevenLabs는 텍스트를 사람 목소리 같은 음성으로 바꾸는 TTS(Text-to-Speech: 글자를 음성으로 읽어 주는 기술) 서비스입니다. 내레이션, 더빙, 캐릭터 음성, 임시 가이드 보이스를 만드는 데 쓰이며, API를 통해 자동으로 음성 파일을 생성하는 흐름에도 연결될 수 있습니다. **대본을 넣으면 읽어 주는 AI 성우실에 가깝습니다.**

음성 복제나 특정 화자 스타일 사용 기능은 제공 여부와 정책이 달라질 수 있습니다. 타인의 목소리를 사용할 때는 동의와 권리 확인이 필요합니다.

쇼츠 내레이션, 정보성 영상 더빙, 다국어 음성 버전 제작, 편집 전 임시 목소리 제작, 자동 콘텐츠 제작 파이프라인에서 사용됩니다. TTS가 글자를 음성으로 바꾸는 기술이라면, STT(Speech-to-Text: 음성을 글자로 바꾸는 기술)는 반대 방향입니다. Suno처럼 음악을 만드는 서비스와도 역할이 다릅니다.

### Suno

Suno는 설명이나 가사를 바탕으로 음악·노래·BGM을 생성하는 AI 음악 서비스입니다. 원하는 분위기, 장르, 가사, 길이 등을 바탕으로 음악을 만들 수 있는 서비스로 알려져 있습니다. **“밝고 빠른 여행 브이로그용 음악”처럼 설명하면 음악 시안을 만들어 주는 AI 작곡가라고 볼 수 있습니다.**

생성 가능한 형식, API 제공 여부, 상업적 이용 조건, 저작권 관련 정책은 변동될 수 있습니다. 유튜브나 광고 영상에 사용할 경우, 업로드 플랫폼의 음악 정책과 Suno의 이용 약관을 함께 확인하는 편이 안전합니다.

오프닝 음악, 엔딩 음악, 배경음악(BGM: 영상 뒤에 깔리는 음악), 쇼츠용 짧은 사운드, 광고 영상의 분위기 시안 제작에 쓰입니다. Suno는 음악 생성 서비스이고, ElevenLabs는 말소리 생성 서비스입니다. 스톡 음악은 이미 만들어진 곡을 라이선스로 구매·사용하는 방식이고, Suno는 새 음악을 생성하는 방식이며, 둘 다 권리 조건 확인이 필요합니다.

## 3.4 영상 후반·조립

영상 후반·조립 도구는 이미 만들어진 영상, 이미지, 음성, 음악을 자르고 붙이고 변환하고 렌더링하는 부류입니다. 생성 AI가 재료를 만든다면, 이 부류의 도구는 그 재료를 실제 영상 파일로 조립하는 역할을 합니다.

### ffmpeg

ffmpeg는 영상·오디오를 자르고, 변환하고, 합치고, 프레임을 추출하는 핵심 명령 도구입니다. 영상 작업에서 가장 널리 쓰이는 기본 일꾼 중 하나이며, 영상 포맷 변환, 해상도 변경, 오디오 추출, 특정 구간 자르기, 여러 클립 합치기, 프레임 이미지 추출, 썸네일 만들기 등에 사용됩니다. **편집실 뒤편에서 파일을 실제로 자르고 포장하는 만능 공구함 같은 도구입니다.**

로컬 AI가 명령 실행 권한을 부여받은 경우, 수십 개 파일을 반복 처리하는 자동화에 매우 유용합니다. 쇼츠 비율 변환, 자막용 오디오 추출, 생성 영상 길이 맞추기, 썸네일용 프레임 뽑기, 여러 클립 이어 붙이기, 업로드용 파일 변환에 쓰입니다.

ffmpeg는 파일 처리 도구이고, Premiere는 사람이 타임라인을 보며 편집하는 프로그램입니다. yt-dlp가 영상을 내려받는 데 강하다면, ffmpeg는 내려받은 파일을 가공하는 데 강합니다. 명령어 옵션은 강력하지만 복잡하므로, 원본 파일을 보존한 상태에서 작업하는 습관이 좋습니다.

### Remotion

Remotion은 React/JavaScript 코드로 영상을 만들고 렌더링하는 도구입니다. 자막, 이미지, 그래프, 텍스트 애니메이션, 반복 템플릿을 코드로 구성해 영상으로 출력할 수 있게 해줍니다. **정해진 영상 양식에 제목, 숫자, 이미지, 대본만 바꿔 넣어 여러 편을 찍어내는 자동 조립 라인에 가깝습니다.**

코딩을 모르는 사람에게는 직접 다루기 어려울 수 있습니다. 다만 로컬 AI나 개발자가 템플릿을 만들어 두면 데이터만 바꿔 여러 영상을 자동 생성하기에 좋고, 특히 같은 형식의 쇼츠나 정보성 영상을 대량 제작할 때 유용합니다.

뉴스 요약 쇼츠, 랭킹 영상, 상품 소개 영상, 자동 자막 영상, 매일 반복되는 데이터 기반 콘텐츠 제작에 쓰입니다. Remotion은 코드 기반 영상 제작 도구이고, Premiere는 타임라인 기반 편집 프로그램입니다. ffmpeg가 파일을 변환하고 합치는 낮은 층위의 도구라면, Remotion은 화면 구성과 애니메이션을 설계해 영상으로 렌더링하는 쪽에 가깝습니다.

### Premiere MCP

Premiere MCP는 Adobe Premiere Pro 같은 편집기를 MCP로 AI 앱에 연결해 편집 작업을 자동화하려는 방식입니다. 하나의 고정된 공식 기능명이라기보다, Premiere와 AI 앱 사이에 MCP 서버나 플러그인, 스크립트 연결을 두는 구성을 가리키는 경우가 많습니다. **Premiere에 AI가 사용할 수 있는 리모컨을 붙여, 가능한 편집 명령을 대신 실행하게 하는 방식입니다.**

지원 여부, 구현 방식, 가능한 편집 명령은 사용 중인 MCP 서버와 Premiere의 확장 기능에 따라 달라집니다. 권한을 부여받은 경우 AI가 클립 불러오기, 파일 정리, 러프컷 배치 같은 작업을 도울 수 있지만, 어떤 기능이 가능한지는 확인이 필요합니다.

러프컷 자동 배치, 클립 정리, 시퀀스 생성, 자막 파일 불러오기, 반복 편집 작업 보조 같은 흐름에서 접할 수 있습니다. Premiere는 편집 프로그램이고, MCP는 AI 앱과 외부 도구를 연결하는 방식입니다. Premiere 자체의 AI 기능, Adobe 플러그인, 커뮤니티 MCP 서버는 서로 다른 층위입니다.

## 3.5 문서·디자인·협업 연동

문서·디자인·협업 도구는 기획, 대본, 자료, 디자인 시안, 작업 상태를 정리하는 부류입니다. 로컬 AI가 API나 연동 권한을 부여받은 경우, 자료를 자동으로 기록하고 상태를 갱신하며 팀 작업 흐름을 정리할 수 있습니다.

### Notion

Notion은 문서, 데이터베이스, 일정, 작업 현황을 함께 관리하는 협업 도구입니다. 아이디어, 대본, 레퍼런스 링크, 업로드 일정, 작업 상태를 정리하는 데 많이 쓰입니다. **콘텐츠 팀의 공용 기획 노트이자 작업 현황판이라고 볼 수 있습니다.**

API를 이용하면 권한이 있는 페이지나 데이터베이스에 새 항목을 만들거나 기존 정보를 수정할 수 있습니다. 로컬 AI가 연결되면 생성한 이미지 목록, 영상 상태, 자막 파일 위치 등을 자동으로 기록하는 흐름을 만들 수 있습니다.

콘텐츠 아이디어 보관함, 대본 관리, 촬영 체크리스트, 편집 상태표, 업로드 캘린더, 협업 피드백 정리에 쓰입니다. Notion 앱, Notion API, Notion AI는 서로 다릅니다. Notion 앱은 사람이 쓰는 작업 공간이고, API는 외부 프로그램이 Notion에 접근하는 통로이며, Notion AI는 Notion 안에서 제공되는 AI 기능입니다.

### Figma

Figma는 디자인과 프로토타입을 만드는 협업 도구입니다. 썸네일 구성, 브랜드 가이드, 화면 디자인, 클릭되는 모형인 프로토타입을 만드는 데 쓰입니다. **디자이너가 화면 구성, 색상, 글꼴, 버튼 모양을 정리해 두는 디자인 작업대에 가깝습니다.**

프로토타입은 사용자가 눌러 볼 수 있는 모형이지만, 실제 기능이 구현된 앱이나 웹사이트는 보통 아닙니다. API나 플러그인을 통해 디자인 파일 정보, 이미지, 컴포넌트 데이터를 가져오는 자동화도 가능합니다.

유튜브 썸네일 템플릿, 채널 브랜딩, 영상 타이틀 카드, 웹페이지 시안, 앱 화면 목업, 디자인 시스템 관리에 쓰입니다. Figma는 디자인 설계와 협업에 강하고, Canva는 템플릿 기반으로 빠르게 완성 이미지를 만드는 데 강합니다. Figma의 프로토타입은 실제 기능이 있는 서비스가 아니라 클릭 흐름을 보여 주는 모형인 경우가 대부분입니다.

### Canva

Canva는 웹 기반 그래픽 디자인 도구로, 썸네일·카드뉴스·배너 등을 템플릿으로 쉽게 만드는 서비스입니다. 디자인 경험이 많지 않아도 템플릿을 활용해 빠르게 결과물을 만들 수 있고, 유튜브 썸네일, SNS 카드, 발표용 이미지, 간단한 영상 디자인까지 폭넓게 사용됩니다. **디자인 초보도 템플릿을 골라 빠르게 결과물을 만드는 온라인 디자인 편집실이라고 볼 수 있습니다.**

API나 앱 연동 기능은 제공 범위가 달라질 수 있으므로, 자동화 목적이라면 2026-05 기준 공식 문서 확인이 필요합니다. 유튜브 썸네일, 쇼츠 커버, 인스타그램 카드뉴스, 광고 배너, 간단한 브랜딩 이미지 제작에 쓰입니다.

Canva는 완성형 마케팅 이미지 제작에 강하고, Figma는 디자인 시스템과 프로토타입 설계에 강합니다. 이미지 생성 모델은 새 이미지를 만들어 내는 도구이고, Canva는 이미지를 배치하고 편집해 최종 디자인으로 만드는 플랫폼입니다.

## 3.6 결과물 확인·관리 (직접 만드는 도구)

AI로 이미지를 수십 장, 영상을 수십 개 만들면 “생성”보다 “고르고 관리하는 일”이 더 커집니다. 이 부류는 외부 서비스라기보다, 로컬 AI와 함께 직접 만들어 쓰는 검수 화면, 갤러리, 대시보드에 가깝습니다.

### HTML 갤러리·대시보드

HTML 갤러리·대시보드는 생성한 이미지·영상·메타데이터를 한 화면에 모아 비교하고 고르기 위해 직접 만드는 웹페이지 형태의 결과물입니다. HTML, CSS, JavaScript를 이용해 로컬 폴더의 이미지, 영상, 프롬프트, 점수, 파일명을 한 화면에 정리할 수 있습니다. **생성된 결과물을 검수하고 관리하기 위한 화면이지, 이미지를 생성하는 모델은 아닙니다.**

로컬 AI가 권한을 부여받은 경우, 생성 결과물을 읽어 자동으로 갤러리 파일을 만들어 줄 수 있습니다. 수백 장의 후보 이미지를 책상 위에 흩어 놓는 대신, 브라우저 안에 정리해 둔 “검수판”이라고 생각하면 쉽습니다.

썸네일 후보 비교, 생성 영상 베스트 컷 선정, 프롬프트별 결과 비교, 파일명·사용 여부·점수 기록, 최종 업로드 후보 정리에 쓰입니다. HTML 갤러리는 편집 도구가 아니라 확인·관리 도구입니다. 실제 자르기와 합치기는 ffmpeg, Premiere, Remotion 같은 도구가 담당합니다. 또한 로컬에서 보는 HTML 대시보드는 인터넷에 공개된 웹사이트와 다릅니다.

### 로컬 HTTP 서버로 띄워 보기

로컬 HTTP 서버는 내 컴퓨터의 폴더를 임시 웹서버로 열어 브라우저에서 결과물을 확인하는 방법입니다. 예를 들어 작업 폴더에서 `python -m http.server`를 실행하면 브라우저에서 `http://localhost:8000` 같은 주소로 HTML 갤러리나 결과물을 확인할 수 있습니다. **완성 전 사이트를 세상에 공개하는 것이 아니라, 내 작업실 안에서만 임시로 켜 보는 미리보기 창에 가깝습니다.**

이 방식은 기본적으로 내 작업 환경에서 확인하기 위한 것이며, 인터넷에 공개 배포하는 것과는 다릅니다. 다만 네트워크 설정, 방화벽, 터널링 도구를 어떻게 쓰느냐에 따라 다른 기기에서 접근 가능해질 수도 있으므로 공개 범위를 확인해야 합니다.

HTML 갤러리 확인, 영상·이미지 미리보기, Remotion 결과물 점검, 자바스크립트가 포함된 대시보드 확인, 생성 결과물 비교 화면을 브라우저에서 볼 때 사용합니다. `localhost`는 보통 내 컴퓨터에서 접속하는 주소입니다. 인터넷 공개 배포는 별도의 호스팅, 서버, 도메인, 보안 설정이 필요한 작업이며, 파일을 더블클릭해 여는 `file://` 방식과 HTTP 서버로 여는 방식은 브라우저 보안 정책 때문에 일부 기능 동작이 다를 수 있습니다.
