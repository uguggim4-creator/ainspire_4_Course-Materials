## 1. 로컬 AI 도구의 종류

여기서 말하는 “로컬 AI”는 AI 모델이 반드시 내 컴퓨터 안에서만 돌아간다는 뜻은 아닙니다. 초급 단계에서는 **내 컴퓨터의 파일·폴더·프로그램과 연결되어, 권한을 부여받은 범위에서 작업을 도와주는 AI**로 이해하시면 됩니다.

챗봇이 답변을 적어주는 도구라면, 로컬 작업 환경의 AI는 원고 폴더를 살피고, 자막 파일을 정리하고, 업로드용 설명문을 만들고, 반복 작업을 자동화하는 흐름까지 이어갈 수 있습니다. 중요한 기준은 제품 이름을 외우는 것이 아니라 **AI가 어느 작업 공간에 붙어 일하는가**입니다.

파일 보기, 파일 수정, 명령 실행, 외부 도구 연결은 모두 사용자가 권한을 부여한 경우에만 가능하다고 이해하는 것이 안전합니다. **구체 기능·가격·지원 운영체제·권한 정책은 2026-05 기준으로도 변동 가능하므로 공식 문서 확인이 필요합니다.**

### 1.1 Claude Desktop

Claude Desktop은 Anthropic의 Claude를 웹브라우저가 아니라 컴퓨터에 설치한 데스크톱 앱에서 사용하는 방식입니다. **권한을 부여받은 파일·도구 연결을 통해 로컬 작업에 붙여 쓸 수 있다는 점이 핵심입니다.**

설정에 따라 Claude Desktop은 MCP(Model Context Protocol: AI 앱이 외부 도구와 주고받는 방법을 정한 연결 규칙)를 통해 파일 시스템이나 다른 도구와 연결될 수 있습니다. 다만 어떤 폴더를 읽고 쓸 수 있는지, 어떤 도구를 연결할 수 있는지는 사용자가 부여한 권한과 앱·운영체제 정책에 따라 달라집니다. 구체 기능, 지원 OS, 요금, 권한 정책은 2026-05 기준 공식 문서 확인이 필요합니다.

쉽게 떠올리면 웹 채팅창에 말만 거는 것이 아니라, 직원에게 특정 책상과 서류함, 업무 도구 사용 권한을 주는 느낌입니다. **사무실 전체가 아니라 허용한 서랍과 도구부터 다룬다고 이해하시면 안전합니다.**

영상 크리에이터는 Claude Desktop을 원고 폴더, 자막 파일, 기획안, 업로드 설명문 템플릿과 함께 쓰는 상황에서 마주칠 수 있습니다. 예를 들어 권한을 부여한 폴더 안의 대본을 요약하거나, 여러 편의 영상 제목 후보를 정리하거나, 자막 파일의 말투를 통일하는 식의 작업에 활용될 수 있습니다. MCP로 연결 가능한 도구가 준비되어 있다면, 단순 대화보다 한 단계 더 실제 작업 흐름에 가까운 방식으로 사용할 수 있습니다.

Claude 웹과 Claude Desktop은 둘 다 Claude 모델을 사용할 수 있지만, Claude Desktop은 데스크톱 앱과 로컬 도구 연결을 중심으로 설정할 수 있다는 점이 다릅니다. Claude 모델은 “두뇌”에 가깝고, Claude Desktop은 그 두뇌를 컴퓨터 작업 공간에 앉히는 “앱”에 가깝습니다. MCP와 API(API: 프로그램끼리 요청과 응답을 주고받는 연결 통로)도 헷갈리기 쉬운데, MCP는 AI 앱이 여러 도구를 붙이기 쉽게 하려는 연결 규칙에 가깝고, API는 서비스끼리 직접 연결할 때 넓게 쓰이는 통신 방식입니다.

### 1.2 Codex

Codex는 OpenAI 계열에서 코드·프로젝트 파일 작업을 맡기기 위한 도구나 환경 이름으로 언급됩니다. **제공 형태는 시기와 문맥에 따라 달라질 수 있으므로 공식 문서 확인이 필요한 항목입니다.**

Codex라는 이름은 문맥에 따라 코드 생성 모델, 코드 작업용 에이전트, CLI(CLI: 명령어를 글자로 입력해 프로그램을 다루는 방식), 웹 또는 IDE(통합 개발 환경: 파일 편집·실행·확인을 한 화면에서 하는 프로그램) 연동 형태로 언급될 수 있습니다. 비개발자에게 중요한 점은 Codex를 “코딩 전용 챗봇”이라기보다 프로젝트 폴더 안의 파일 수정, 오류 확인, 반복 작업 자동화를 맡기는 작업 환경 후보로 이해하는 것입니다. 웹에서 실행되는지, 내 컴퓨터의 터미널(명령어 입력 창)에서 실행되는지, IDE에 붙는지는 시기와 제공 방식에 따라 달라질 수 있으며, 파일 접근과 명령 실행은 권한을 부여받은 범위에서만 가능하다고 보는 것이 안전합니다.

비유하자면 직원에게 “이 폴더의 자동화 작업을 정리해 주세요”, “오류가 왜 나는지 봐 주세요”, “반복 작업을 줄이는 스크립트를 만들어 주세요”라고 맡기는 느낌입니다. **다만 그 직원이 웹에서 일하는지, 내 컴퓨터 안의 명령어 창에서 일하는지, 기존 편집기 안에서 일하는지는 공식 문서로 확인해야 합니다.**

영상 크리에이터는 Codex를 자막 변환, 파일 이름 일괄 정리, 쇼츠 클립 목록 생성, 업로드 메타데이터 CSV 정리, ffmpeg(ffmpeg: 영상·오디오 변환과 처리를 명령어로 수행하는 도구) 명령 작성 같은 자동화 작업에서 마주칠 수 있습니다. 특히 내 영상 제작 과정을 반복 가능한 프로젝트로 만들고 싶을 때, 코드와 파일을 함께 다루는 도구가 필요해질 수 있습니다. 코딩을 직접 잘 몰라도 오류 메시지를 설명받거나, 기존 자동화 파일을 수정하는 보조 역할로 사용할 수 있습니다.

Codex와 ChatGPT는 둘 다 OpenAI 계열로 언급될 수 있지만, ChatGPT가 대화 중심으로 익숙하다면 Codex는 코드·파일 작업 환경 쪽으로 이해하는 것이 좋습니다. Codex와 Claude Desktop은 제공사와 중심 용도가 다르며, 어느 쪽이 더 좋다기보다 어떤 작업 공간에 붙여 쓸지의 차이로 보는 편이 안전합니다. Codex와 VS Code 익스텐션도 구분해야 하는데, Codex는 특정 도구나 환경 이름으로 언급되는 반면 VS Code 익스텐션은 기존 편집기에 AI 기능을 붙이는 사용 방식입니다.

### 1.3 Antigravity

Antigravity는 AI IDE 또는 프로젝트 작업 환경으로 언급되는 도구명입니다. **설치 방식과 로컬 권한은 공식 문서 확인이 특히 필요한 항목입니다.**

AI IDE는 인공지능이 결합된 통합 개발 환경으로, 프로젝트 파일, AI 채팅, 실행 결과, 터미널 등을 한 화면에서 다루도록 설계된 작업 공간을 뜻합니다. Antigravity는 이런 AI IDE 또는 프로젝트 작업 환경으로 언급될 수 있으나, 제품 변경 가능성이 큰 영역입니다. 따라서 설치 방식, 지원 운영체제, 로그인 계정, 파일 접근 범위, 명령 실행 지원 여부를 공식 문서에서 확인해야 합니다. 영상 크리에이터 입장에서는 내 작업 자동화 프로젝트를 통째로 관리하는 작업실 후보로 볼 수 있지만, 실제 기능은 2026-05 기준 공식 문서 확인이 필요합니다.

쉽게 말하면 새 편집실을 통째로 빌리는 느낌입니다. **그 안에 AI 직원, 파일장, 실행 버튼, 확인 화면이 함께 있을 수 있지만, 어떤 장비가 포함되는지는 공식 안내를 보고 확인해야 합니다.**

Antigravity 같은 AI IDE형 도구는 영상 제작 자동화 프로젝트를 하나의 작업실처럼 관리하고 싶을 때 등장할 수 있습니다. 예를 들어 자료 수집, 원고 생성, 자막 정리, 영상 처리 명령, 업로드 전 검수 목록을 한 프로젝트 폴더 안에서 관리하는 식입니다. 단, 실제로 파일을 읽거나 수정하는지, 명령을 실행할 수 있는지, 외부 서비스와 연결되는지는 권한 설정과 제품 정책에 따라 달라집니다.

Antigravity와 AI IDE는 구분해야 합니다. AI IDE는 도구의 “종류”이고, Antigravity는 그 종류로 언급되는 “제품명”입니다. Antigravity와 VS Code 익스텐션도 다릅니다. Antigravity는 별도의 프로젝트 작업 환경으로 제공될 수 있는 반면, VS Code 익스텐션 방식은 이미 있는 VS Code 안에 AI 기능을 붙이는 방식입니다. 또한 AI IDE라고 해서 반드시 AI 모델이 내 컴퓨터 안에서만 실행된다는 뜻은 아니므로, 로컬 파일 접근과 모델 실행 위치를 따로 확인해야 합니다.

### 1.4 VS Code 익스텐션으로 쓰는 방식

VS Code 익스텐션 방식은 Visual Studio Code 안에 AI 확장을 설치해 쓰는 방식입니다. **기존 편집기에서 파일·프로젝트 작업을 보조받는 사용 방식이라고 이해하시면 됩니다.**

VS Code(Visual Studio Code: 무료로 많이 쓰이는 코드 편집기)는 본래 개발자가 파일을 열고 수정하는 도구입니다. 익스텐션(확장: 기본 프로그램에 기능을 추가하는 작은 부가 도구)을 설치하면 AI 채팅, 코드 수정 제안, 프로젝트 검색, 터미널 명령 안내 같은 기능을 같은 화면에서 사용할 수 있습니다. 이 방식은 Claude Desktop, Codex, Antigravity처럼 하나의 독립 도구를 말한다기보다, 기존 편집기 안에 AI 기능을 붙이는 사용 방식입니다. 어떤 모델을 쓰는지, 파일 접근이나 명령 실행 권한이 있는지, 결제와 로그인 방식이 무엇인지는 각 익스텐션별로 2026-05 기준 공식 문서 확인이 필요합니다.

비유하면 이미 쓰는 공책에 AI 포스트잇을 붙이는 느낌입니다. **공책은 VS Code이고, 포스트잇 역할을 하는 것이 AI 익스텐션입니다.**

영상 크리에이터는 프롬프트 모음, 대본 템플릿, 자막 처리 파일, 자동화 스크립트, 업로드 설명문 양식을 한 폴더에 모아 관리할 때 VS Code를 만날 수 있습니다. 여기에 AI 익스텐션을 붙이면 권한을 부여한 작업 폴더 안에서 파일을 설명받거나, 수정 제안을 받거나, 반복 작업용 코드를 만드는 도움을 받을 수 있습니다. 코딩을 전문적으로 하지 않더라도 내 영상 제작용 작업 폴더를 체계적으로 관리하는 도구로 접근할 수 있습니다.

VS Code와 VS Code 익스텐션은 다릅니다. VS Code는 기본 편집기이고, 익스텐션은 그 위에 추가하는 기능입니다. VS Code 익스텐션 방식과 로컬 모델도 다릅니다. 익스텐션을 쓴다고 해서 AI 모델이 반드시 내 컴퓨터 안에서 실행되는 것은 아니며, 클라우드 모델을 호출할 수도 있습니다. 또한 익스텐션이 MCP를 지원할 수는 있지만, 익스텐션 자체가 곧 MCP라는 뜻은 아닙니다.

### 1.5 한눈에 비교 (어떤 상황에 무엇을)

여러 로컬 AI 도구를 볼 때는 성능 순서보다 작업 공간을 기준으로 나누는 편이 이해하기 쉽습니다. **핵심은 AI를 어느 작업 공간에 붙이는가입니다.**

로컬 AI 도구를 고를 때 첫 질문은 “어느 모델이 가장 좋은가”보다 “내 작업 파일이 어디에 있고, AI에게 어디까지 권한을 줄 것인가”입니다. Claude Desktop, Codex, Antigravity는 각각 이름을 가진 도구나 환경으로 이해하면 되고, VS Code 익스텐션은 기존 편집기 안에 AI를 붙이는 사용 방식입니다. 아래 비교는 성능 순위가 아니라 역할과 상황을 나눈 것이며, 2026-05 기준 공식 문서 확인이 필요합니다.

쉽게 말하면 직원을 뽑는 문제라기보다, 직원이 일할 자리를 어디에 마련할지 정하는 문제입니다. **채팅방에 앉힐지, 프로젝트 작업실에 앉힐지, 기존 편집 책상 옆에 붙일지의 차이입니다.**

가장 많이 헷갈리는 부분은 “도구”와 “사용 방식”입니다. Claude Desktop, Codex, Antigravity는 특정 이름으로 불리는 도구나 환경에 가깝고, VS Code 익스텐션은 VS Code 안에 AI 기능을 붙이는 방식입니다. 또 “로컬 작업 환경”과 “로컬 모델”도 다릅니다. 내 컴퓨터의 파일과 연결된다고 해서 AI 모델이 반드시 내 컴퓨터 안에서만 실행된다는 뜻은 아니므로, 모델 실행 위치와 파일 접근 권한을 따로 확인해야 합니다.
