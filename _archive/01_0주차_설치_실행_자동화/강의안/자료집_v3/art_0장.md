## 0. 들어가며 — 왜 로컬 AI인가

챗봇으로 AI를 쓰는 사람은 이미 많습니다. 브라우저(웹사이트를 여는 프로그램)나 앱에서 ChatGPT, Claude, Gemini 같은 AI에게 질문하고, 답을 받고, 그 답을 복사해 글이나 기획서에 옮기는 방식입니다. 이 방식만으로도 아이디어를 얻고, 문장을 다듬고, 제목 후보를 뽑는 데 큰 도움이 됩니다.

그런데 영상 제작자의 일은 대화창 안에서 끝나지 않습니다. 레퍼런스 링크, 촬영본, 자막 파일, 음성 파일, 썸네일 이미지, 업로드 문구, 폴더 정리까지 대부분의 작업물이 내 컴퓨터 어딘가에 있습니다. **AI가 “좋은 답변”만 해주는 단계를 넘어 실제 파일과 작업 흐름을 다룰 수 있다면, 할 수 있는 일이 훨씬 많아집니다.**

이 자료집에서 말하는 로컬 AI는 반드시 “인터넷 없이 내 컴퓨터 안에서만 돌아가는 AI 모델”만을 뜻하지 않습니다. 더 넓게는 **내 컴퓨터를 작업 공간으로 삼아, 권한을 부여받은 범위 안에서 파일을 읽고, 만들고, 정리하고, 프로그램을 실행할 수 있는 AI 도구**를 말합니다. 도구에 따라서는 클라우드 모델(인터넷상의 서버에서 계산을 처리하는 AI 모델)을 함께 사용할 수도 있습니다.

핵심 변화는 단순합니다. 챗봇 AI가 “대답하는 AI”에 가깝다면, 로컬 AI는 “일을 처리하는 AI”에 가깝습니다. 마치 내 직원에게 일을 맡기듯 자료 수집, 정리, 생성, 저장 같은 여러 단계를 이어서 맡길 수 있습니다. 물론 이것은 비유입니다. **AI가 마음대로 모든 것을 하는 것이 아니라, 사용자가 허용한 권한과 승인 범위 안에서 작동한다는 점이 중요합니다.**

### 0.1 챗봇으로 쓰는 AI vs 로컬로 쓰는 AI

챗봇으로 쓰는 AI는 대화창에서 답을 받는 방식입니다. 브라우저나 앱의 대화창에 “이 영상 제목을 20개 추천해 줘”라고 입력하면, AI가 제목 후보를 텍스트로 답합니다. 그다음에는 사용자가 답변을 복사해서 메모장, 문서, 스프레드시트, 편집 프로그램 등에 직접 옮겨야 합니다. **챗봇 방식은 좋은 답을 받는 데 강하지만, 그 답을 실제 작업물에 반영하는 일은 보통 사람이 이어서 처리합니다.**

로컬로 쓰는 AI는 작업 위치가 대화창 안에만 머물지 않습니다. 로컬 AI 도구가 지원하고 사용자가 권한을 허용하면, 특정 폴더의 파일을 읽고 결과물을 새 파일로 저장하거나 필요한 프로그램과 명령을 실행할 수 있습니다. 여기서 권한은 프로그램이 특정 파일, 폴더, 앱, 계정에 접근해도 된다고 사용자가 허용하는 범위를 뜻합니다. **즉 로컬 AI는 권한을 부여받은 경우 내 컴퓨터의 파일과 프로그램을 다루며 실제 작업을 이어서 수행하는 방식입니다.**

예를 들어 챗봇 방식에서는 “이 원고를 요약해 줘”라고 묻고, 나온 답변을 사람이 복사해야 합니다. 로컬 AI 방식에서는 권한을 허용한 작업 폴더 안에서 “이 폴더의 원고 파일들을 모두 읽고, 영상별 요약본을 만들어 별도 폴더에 저장해 줘”처럼 요청할 수 있습니다. 다만 모든 도구가 이런 기능을 지원하는 것은 아니므로, 도구별 기능과 권한 설정을 확인해야 합니다.

쉽게 말해 챗봇 AI는 메신저로 조언을 주는 전문가에 가깝습니다. 로컬 AI는 허락받은 책상, 서랍, 파일철을 열어 실제 정리 업무까지 도와주는 보조 직원에 가깝습니다. **차이는 “답을 주는가”에서 끝나는지, 아니면 허용된 작업 공간 안에서 파일과 결과물까지 다루는지에 있습니다.**

실제 활용에서는 이런 차이가 더 분명해집니다:

- 챗봇 방식은 제목 아이디어, 대본 초안, 문장 수정, 기획 방향 상담처럼 “답변을 받아 사람이 옮기는 일”에 자주 쓰입니다.
- 로컬 방식은 폴더 안 파일을 읽어 정리하기, 여러 개의 결과물을 한꺼번에 만들기, 파일 이름 바꾸기, 표 파일로 저장하기, 반복 작업 자동화에 쓰입니다.
- 영상 제작에서는 대본, 자막, 썸네일 문구, 레퍼런스 목록, 업로드 설명문처럼 여러 파일로 흩어지는 작업을 한 흐름으로 묶을 때 차이가 크게 느껴집니다.

비슷한 표현은 다음처럼 나누어 보면 이해하기 쉽습니다:

| 구분 | 어떻게 다른가 |
|---|---|
| 로컬 AI 도구 vs 로컬 AI 모델 | 로컬 AI 모델은 AI 모델 자체가 내 컴퓨터에서 실행되는 경우를 뜻합니다. 로컬 AI 도구는 내 컴퓨터의 파일과 프로그램을 작업 대상으로 삼는 도구를 뜻하며, 경우에 따라 클라우드 모델을 함께 사용할 수 있습니다. |
| 데스크톱 앱 vs 로컬 AI | 어떤 AI 서비스의 데스크톱 앱을 설치했다고 해서 자동으로 로컬 AI가 되는 것은 아닙니다. 내 파일이나 프로그램에 접근할 권한을 받고 실제 작업을 수행할 수 있어야 로컬 작업 도구에 가깝습니다. |
| 파일 업로드 vs 폴더 접근 | 챗봇에 파일 하나를 올려 분석받는 것과, 로컬 AI가 권한을 받은 폴더 안의 여러 파일을 읽고 정리하는 것은 다릅니다. 후자는 작업 범위와 자동화 가능성이 더 넓습니다. |

### 0.2 로컬 AI가 열어주는 가능성 — “내 직원처럼 일하는 AI”

로컬 AI가 “내 직원처럼 일한다”는 말은 AI가 사람처럼 책임을 진다는 뜻이 아닙니다. **권한과 승인 범위 안에서 자료 수집, 정리, 생성, 저장 같은 여러 단계를 이어서 처리할 수 있다는 뜻입니다.** 챗봇 AI가 보통 한 번의 질문에 한 번의 답변을 준다면, 로컬 AI 도구는 기능과 권한이 허용되는 경우 목표를 받아 여러 단계를 이어서 수행할 수 있습니다. 예를 들어 “이 폴더의 인터뷰 대본을 읽고, 핵심 발언을 뽑고, 쇼츠 아이디어 30개를 만든 뒤, CSV 파일로 저장해 줘”처럼 요청할 수 있습니다. CSV는 엑셀이나 스프레드시트에서 열 수 있는 표 형식 파일입니다.

“내 직원처럼”이라는 표현은 사용자가 일을 지시하고, 작업 범위를 정하고, 중요한 결정은 승인하는 구조에 더 가깝습니다. 좋은 직원에게 업무 매뉴얼과 접근 권한을 주듯이, 로컬 AI에도 어떤 폴더를 다뤄도 되는지, 어떤 결과물을 원하는지, 어디까지 자동으로 해도 되는지를 알려주어야 합니다. **지능이 높아지는 것과 권한이 자동으로 생기는 것은 별개의 문제입니다.**

작업 환경이 정리되어 있을수록 로컬 AI가 할 수 있는 일도 늘어납니다. 폴더 이름, 파일 이름, 결과물 저장 위치, 승인 규칙이 분명하면 AI가 처리할 수 있는 범위가 넓어집니다. 반대로 자료가 뒤섞여 있거나 지시가 모호하면, AI가 만든 결과를 사람이 다시 정리해야 할 수 있습니다. **로컬 AI를 잘 쓰는 일은 AI에게 일을 맡기기 전에 작업 공간을 알아보기 쉽게 만드는 일과도 연결됩니다.**

챗봇 AI가 “이렇게 하면 좋겠습니다”라고 말하는 조언자라면, 로컬 AI는 “말씀하신 폴더에서 자료를 찾아 정리해 두었습니다”라고 보고하는 보조 직원에 가깝습니다. 다만 회사 규칙과 권한이 없는 직원이 마음대로 결제하거나 업로드할 수 없듯, AI도 허용된 범위 안에서만 움직여야 합니다.

영상 제작 흐름에서는 대표적으로 이런 일들에 활용할 수 있습니다:

- 자료 수집에서는 특정 주제의 레퍼런스 링크, 제목, 썸네일 문구, 조회 가능한 공개 정보를 정리할 수 있습니다. 단, 웹사이트 이용 약관과 저작권 범위 안에서 다뤄야 합니다.
- 정리에서는 여러 원고, 자막, 메모 파일을 읽고 주제별로 분류하거나 요약할 수 있습니다.
- 생성에서는 제목 후보, 썸네일 문구, 영상 설명란, 쇼츠 아이디어, 이미지 생성용 프롬프트를 만들 수 있습니다. 프롬프트는 AI에게 원하는 결과를 설명하는 지시문입니다.
- 저장에서는 결과를 문서, 표, 마크다운 파일, 폴더 구조 등으로 저장할 수 있습니다. 마크다운은 제목과 목록을 간단한 기호로 표시하는 문서 형식입니다.
- 반복 처리에서는 파일 이름 바꾸기, 자막 포맷 정리, 결과물 목록 만들기처럼 사람이 반복해서 눌러야 했던 일을 줄일 수 있습니다.

비슷해 보이지만 꼭 구분해야 하는 지점도 있습니다:

| 구분 | 어떻게 다른가 |
|---|---|
| 자동화 vs 완전 방치 | 자동화는 반복 작업을 정해진 순서로 처리하게 만드는 것입니다. 완전히 검토 없이 방치한다는 뜻은 아닙니다. |
| 지능 vs 권한 | AI가 똑똑하다고 해서 자동으로 내 파일을 읽을 수 있는 것은 아닙니다. 파일 접근, 프로그램 실행, 계정 연결은 별도의 권한이 필요합니다. |
| 생성 vs 실행 | AI가 업로드 문구를 생성하는 것과 실제로 SNS에 업로드하는 것은 다릅니다. 실제 게시, 삭제, 결제, 외부 발송 같은 작업은 승인 단계를 두는 것이 안전합니다. |

### 0.3 영상 크리에이터에게 로컬 AI가 특히 강력한 이유

영상 크리에이터의 업무는 대량의 파일, 반복 작업, 여러 도구 사이의 이동이 많습니다. 그래서 로컬 AI는 자료 정리부터 생성, 편집 보조, 업로드 준비까지 큰 도움을 줄 수 있습니다. **영상 제작의 많은 과정이 결국 파일과 폴더를 중심으로 움직이기 때문입니다.**

영상 제작은 창의적인 일처럼 보이지만, 실제로는 반복적이고 양이 많은 작업이 매우 많습니다. 레퍼런스를 찾고, 대본을 정리하고, 자막을 맞추고, 썸네일 후보를 만들고, 결과물을 비교하고, 업로드 문구를 준비하는 과정이 계속 이어집니다. 이 작업 흐름, 즉 워크플로는 대부분 파일과 폴더를 중심으로 움직입니다.

로컬 AI가 강력한 이유는 바로 이 지점에 있습니다. 권한을 허용한 작업 폴더 안에서 AI가 여러 파일을 읽고, 필요한 결과물을 만들고, 정해진 위치에 저장할 수 있다면 제작자는 반복 작업보다 판단과 선택에 더 많은 시간을 쓸 수 있습니다. **로컬 AI는 창작자의 최종 감각을 대신한다기보다, 창작자가 고를 수 있는 선택지를 빠르게 준비해 주는 역할에 가깝습니다.**

특히 영상 크리에이터는 텍스트, 이미지, 음성, 영상이 모두 섞인 작업을 합니다. 로컬 AI 도구가 지원하는 범위 안에서는 자막 파일, 음성 인식 결과, 이미지 후보, 영상 클립 정보, 업로드 문구를 한 프로젝트 단위로 묶어 다루기 좋습니다. 그래서 단순히 “대본을 잘 써주는 AI”보다 “내 프로젝트 폴더 안에서 일을 정리해 주는 AI”의 가치가 크게 느껴질 수 있습니다.

쉽게 말하면 영상 크리에이터는 감독이자 PD이고, 로컬 AI는 자료 조사원, 조연출, 편집 보조, 업로드 매니저를 조금씩 맡는 보조 직원에 가깝습니다. **최종 선택은 사람이 하되, 고를 후보와 정리된 자료를 AI가 미리 준비해 두는 구조입니다.**

구체적으로는 이런 장면에서 자주 쓰입니다:

- 레퍼런스 영상 대량 수집에서는 공개적으로 접근 가능한 범위에서 레퍼런스 링크, 제목, 썸네일 문구, 주제, 포맷을 표로 정리할 수 있습니다. 플랫폼 약관과 저작권을 확인하는 전제가 필요합니다.
- 이미지·영상 생성 보조에서는 썸네일 콘셉트, 이미지 생성 프롬프트, 영상 생성 프롬프트, 변형안 목록을 만들고 결과물을 폴더별로 정리할 수 있습니다.
- 자막·음성 처리에서는 음성을 글자로 바꾸는 STT, 글자를 음성으로 바꾸는 TTS 도구와 연결해 대본, 자막, 내레이션 파일을 정리할 수 있습니다. 도구별 지원 여부는 확인이 필요합니다.
- 결과물 셀렉용 갤러리 제작에서는 여러 썸네일 후보나 이미지 결과물을 한 화면에서 비교할 수 있는 간단한 갤러리 파일을 만들 수 있습니다. 예를 들어 브라우저에서 열리는 HTML 목록 페이지 형태가 될 수 있습니다.
- SNS 업로드 준비에서는 제목, 설명란, 해시태그, 고정 댓글, 짧은 소개문을 플랫폼별로 나눠 준비할 수 있습니다. 실제 자동 업로드는 플랫폼 API(서비스가 외부 프로그램과 연결되도록 제공하는 공식 통로), 계정 권한, 정책이 허용되는 경우에만 가능합니다.
- 프로젝트 관리에서는 촬영본, 편집본, 썸네일, 자막, 업로드 문구를 프로젝트별 폴더로 정리하고 누락된 파일을 확인하는 데 쓸 수 있습니다.

마지막으로, 로컬 AI를 영상 제작에 활용할 때도 비슷한 개념을 구분해 두면 좋습니다:

| 구분 | 어떻게 다른가 |
|---|---|
| AI 영상 생성 vs 영상 제작 자동화 | AI 영상 생성은 새로운 영상이나 이미지를 만들어 내는 일에 가깝습니다. 영상 제작 자동화는 레퍼런스 수집, 파일 정리, 자막 처리, 업로드 준비처럼 제작 전후의 흐름을 줄이는 일까지 포함합니다. |
| 편집을 대신함 vs 편집 보조 | 로컬 AI가 모든 편집 감각을 대신한다기보다는, 컷 목록 정리, 자막 초안, 썸네일 후보, 설명문 작성처럼 편집자가 판단하기 전의 준비 작업을 줄여 줍니다. |
| 업로드 가능 vs 항상 허용 | 어떤 도구가 업로드 기능을 제공하더라도, 각 플랫폼의 정책, API 권한, 계정 설정에 따라 가능 여부가 달라질 수 있습니다. 공개 게시 전에는 승인 단계를 두는 것이 좋습니다. |

### 참고

서비스 정책과 권한 관련 정보는 바뀔 수 있으므로, 실제로 사용하기 전에는 아래 항목을 확인하는 편이 좋습니다:

- ChatGPT, Claude, Gemini 등 개별 서비스의 파일 처리, 데스크톱 앱, 커넥터, 데이터 사용 정책은 바뀔 수 있습니다. (2026-05 기준, 공식 문서 확인 필요)
- 로컬 AI 도구의 파일 접근, 명령 실행, 외부 계정 연결 권한은 도구와 운영체제 설정에 따라 달라집니다. 중요한 작업 폴더와 민감한 파일은 권한 범위를 나누어 관리하는 것이 좋습니다.
- 로컬 AI에게 맡길 작업은 “읽기만 가능”, “새 파일 생성 가능”, “기존 파일 수정 가능”, “외부 서비스 업로드 가능”처럼 권한 단계를 나누어 생각하면 좋습니다.
- 삭제, 덮어쓰기, 공개 업로드처럼 되돌리기 어려운 작업은 사용 중인 도구의 승인 기능과 로그 기록 기능을 확인하는 것이 좋습니다. (2026-05 기준, 공식 문서 확인 필요)
- YouTube, Instagram, TikTok 등 플랫폼의 자동 업로드, 예약 게시, 데이터 접근 정책은 변동될 수 있습니다. (2026-05 기준, 공식 API 문서와 이용 약관 확인 필요)
- 저작권, 초상권, 음원 사용권은 자동화 여부와 관계없이 확인해야 합니다. 로컬 AI는 작업을 빠르게 해줄 수 있지만, 공개 콘텐츠의 책임과 최종 판단은 제작자에게 남습니다.
