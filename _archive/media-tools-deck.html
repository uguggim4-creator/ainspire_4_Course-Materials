AI는 각 단계마다 다른 도구를 호출합니다. 명령어 하나하나를 외울 필요는 없습니다. 어떤 단계에서 어떤 도구가 왜 쓰이는지를 이해하는 것이 핵심입니다.
Step 1 / yt-dlp
yt-dlp — 소스 구하기
도구 소개
yt-dlp는 YouTube를 비롯한 수백 개 온라인 플랫폼에서 영상·오디오를 내려받는 오픈소스 명령줄 도구입니다. 화질·포맷 선택, 오디오만 추출, 자막 파일 다운로드까지 폭넓게 지원합니다.
활용 방식
영상 제작 파이프라인에서 가장 먼저 실행되는 단계입니다. 레퍼런스 클립을 가져오거나, BGM 소스를 수집하거나, AI 파이프라인에 넣을 원본 영상을 확보할 때 활용합니다. 별도 설치 없이 pip install yt-dlp 한 줄로 사용할 수 있습니다.
이렇게 요청하세요
"yt-dlp를 활용해서 이 유튜브 영상 원본이랑 음성만 따로 받아줘."
이런 결과를 받습니다
원본 영상 파일(mp4)과 오디오 파일(mp3)이 각각 저장되어, 이후 단계의 작업 소스로 바로 활용할 수 있습니다.
1. 요청 → 실행
2. 결과물 — 내려받은 원본 영상
예시 영상: Big Buck Bunny (c) 2008 Blender Foundation — CC BY 3.0 / bigbuckbunny.org
Step 2 / FFmpeg
FFmpeg — 자르고 다듬기
도구 소개
FFmpeg는 영상·오디오 파일을 자르고, 붙이고, 변환하는 오픈소스 멀티미디어 처리 도구입니다. 포맷 변환(mp4 / mov / webm), 구간 자르기, 해상도 조정, 화면 크롭을 명령어 한 줄로 처리할 수 있습니다.
활용 방식
쇼츠·릴스는 9:16 세로 비율이 기본입니다. 원본이 16:9 가로 영상이라면 이 단계에서 크롭과 리사이즈를 거칩니다. AI 파이프라인에서 전처리 단계로 가장 빈번히 호출됩니다.
이렇게 요청하세요
"FFmpeg로 이 영상에서 0분 10초부터 25초까지만 잘라서 9:16 세로 비율로 만들어줘."
이런 결과를 받습니다
쇼츠 규격(1080x1920)에 맞게 잘리고 크롭된 세로 영상 클립이 생성됩니다. 바로 다음 단계에 사용할 수 있는 상태로 저장됩니다.
1. 요청 → 실행
2. 결과물 — 9:16 세로로 변환된 클립
Step 3 / Whisper
Whisper — 자동 자막 생성
도구 소개
Whisper는 OpenAI가 공개한 오픈소스 음성 인식(STT) 모델입니다. 오디오 파일을 입력하면 텍스트와 타임스탬프가 포함된 .srt 자막 파일을 출력합니다. 한국어를 포함한 99개 언어를 지원합니다.
활용 방식
쇼츠·릴스에서 자막은 선택이 아닌 필수입니다. 시청자 대부분이 소리 없이 영상을 시청하기 때문입니다. Whisper로 자막을 자동 생성하면 수작업 없이 타임스탬프가 정확한 자막 파일을 얻을 수 있습니다.
이렇게 요청하세요
"Whisper로 이 영상 음성을 받아써서 자막 파일로 만들어줘."
이런 결과를 받습니다
각 문장의 시작·끝 타임코드가 들어간 자막 파일(.srt)이 자동 생성됩니다. 별도 편집 없이 바로 영상에 얹을 수 있는 상태로 출력됩니다.
1. 요청 → 실행
2. 결과물 — 생성된 자막 파일(.srt)
Step 4 / FFmpeg
FFmpeg — 조립·마무리
도구 소개
2단계에서 전처리에 쓰인 FFmpeg가 이번에는 최종 조립 단계에서 다시 등장합니다. 자막 굽기(burn-in), BGM 믹싱, 클립 이어붙이기, 최종 mp4 인코딩을 모두 처리합니다.
활용 방식
Whisper로 만든 .srt 자막, 별도로 준비한 BGM, 편집된 영상 클립을 하나의 완성 파일로 합치는 파이프라인의 종착점입니다. AI가 모든 재료를 준비한 뒤 마지막으로 호출하는 도구입니다.
이렇게 요청하세요
"FFmpeg로 편집한 영상에 이 자막이랑 BGM 합쳐서 최종 mp4로 뽑아줘."
이런 결과를 받습니다
자막이 화면에 입혀지고 BGM이 적절한 볼륨으로 믹싱된 업로드용 최종 영상 파일이 완성됩니다.
1. 요청 → 실행
2. 결과물 — 자막 입힌 최종 쇼츠
전체 흐름 다시보기
한눈에 보는 쇼츠 제작 파이프라인
Step 1 -> yt-dlp -> YouTube 등에서 영상·오디오 파일을 내려받습니다.
Step 2 -> FFmpeg -> 구간 자르기, 포맷 변환, 9:16 세로 크롭을 수행합니다.
Step 3 -> Whisper -> 음성을 텍스트로 변환해 타임스탬프가 담긴 .srt 파일을 만듭니다.
Step 4 -> FFmpeg -> 자막 굽기·BGM 믹스 후 최종 mp4로 인코딩합니다.
각 단계는 독립적으로 실행되며, AI는 상황에 따라 필요한 단계만 선택적으로 호출할 수 있습니다.
부록 — 용어 정리
알아두면 좋은 핵심 용어
코덱 vs 컨테이너
컨테이너(.mp4, .mov)는 파일 포장 박스이고, 코덱(H.264, AAC)은 그 안에 담긴 압축 방식입니다. 같은 .mp4라도 코덱이 다를 수 있습니다.
비트레이트
1초에 처리하는 데이터 양(kbps/Mbps)입니다. 높을수록 화질이 좋고 파일 크기도 커집니다. 플랫폼마다 권장 비트레이트가 있습니다.
fps (프레임 레이트)
1초에 보여주는 정지 화면 수입니다. 24fps는 영화 감성, 30fps는 일반 영상, 60fps는 부드러운 동작에 씁니다. 쇼츠는 대체로 30fps가 기본입니다.
해상도 / 종횡비
해상도는 픽셀 수(예: 1920x1080)이고, 종횡비는 가로:세로 비율입니다. 16:9는 가로 영상(유튜브), 9:16은 세로 영상(쇼츠·릴스)입니다.
STT
Speech-To-Text의 약자입니다. 음성을 텍스트로 변환하는 기술이며, Whisper가 이 역할을 담당합니다. 반대 방향은 TTS(Text-To-Speech)입니다.
렌더링
코드·설정·레이어를 계산해 실제 픽셀로 된 영상 파일을 만들어내는 과정입니다. 계산량이 많아 시간이 걸리며, Remotion의 CLI 명령이 이 역할을 합니다.
보너스 / Remotion
Remotion — 코드로 영상 만들기
도구 소개
Remotion은 React 컴포넌트로 영상의 각 프레임을 정의하고 mp4로 렌더링하는 프로그래매틱 영상 생성 프레임워크입니다. 브라우저에서 실시간 미리보기를, CLI에서 최종 렌더링을 수행합니다.
활용 방식
AI가 자막 타이밍 데이터를 JSON으로 생성했을 때, 이를 애니메이션 자막 영상으로 자동 변환하는 데 적합합니다. 같은 템플릿으로 수십 편의 영상을 일괄 생성하는 자동화 시나리오에서 특히 유용합니다.
이렇게 요청하세요
"Remotion으로 이 자막 텍스트가 움직이는 자막 영상을 만들어줘."
"Remotion으로 조회수 데이터를 막대그래프 영상으로 자동 생성해줘."
이런 결과를 받습니다
입력한 데이터와 텍스트가 반영된 영상이 코드로 자동 렌더링됩니다. 같은 템플릿으로 내용만 바꿔 반복 제작하기가 쉬워집니다.
코드
export const MyVideo = () => {
  const frame = useCurrentFrame();
  const opacity = interpolate(
    frame, [0, 30], [0, 1]
  );
  return (
    <AbsoluteFill>
      <h1 style={{ opacity }}>
        조회수 128,400회
      </h1>
    </AbsoluteFill>
  );
};
